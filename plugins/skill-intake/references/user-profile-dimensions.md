# ユーザープロファイル 6 軸

非技術者を含む不特定多数を相手にするため、最初の数発話でユーザー像を 6 軸で推定する。推定結果は `intake.json` の `user_profile` に格納され、後続 SubAgent の語彙難易度・図解選択・選択肢提示の粒度に反映される。

## 6 軸サマリ

| # | 軸 | レンジ | 主な判定材料 |
|---|----|--------|--------------|
| 1 | 熟練度 | 非技術／中級／上級 | 用語使用・質問粒度 |
| 2 | 役割 | 個人事業主／会社員／経営者／クリエイター | 自己紹介・目的の主語 |
| 3 | 文脈 | 業務／個人／学習／趣味 | 利用シーンの主体 |
| 4 | 制約 | 時間／予算／権限／知識 | 「できない理由」の語り口 |
| 5 | 動機 | 効率化／品質／学習／ブランディング | 「何が浮く？」への回答 |
| 6 | 共有意図 | 自分のみ／少人数／不特定多数／顧客 | 成果物の届け先 |

---

## 1. 熟練度（technical_level）

| レベル | 判定シグナル | 後続への影響 |
|--------|--------------|--------------|
| 非技術 | 「API って何？」「自動でやってほしい」抽象要望 | 専門用語完全禁止／非技術者言い換え必須／★1 図種禁止 |
| 中級 | サービス名は知っている／設定画面は触れる／コードは読まない | 用語に括弧で簡易説明／Mermaid 採用可 |
| 上級 | API/JSON/CLI を自然に使う／コード例を望む | 専門用語 OK／詳細スキーマ提示 OK |

**用語使用シグナル例**

- 「フォームって作れる？」→ 非技術
- 「Google Forms と Sheets を連携したい」→ 中級
- 「OAuth スコープ form.body の prefill で …」→ 上級

---

## 2. 役割（role）

| 役割 | シグナル | 留意点 |
|------|----------|--------|
| 個人事業主 | 「私が／一人で」自分 = 実行者 = 決裁者 | 即時利用前提・運用コスト最小化 |
| 会社員 | 「上司に／部署で」承認フロー言及 | 共有相手・社内ツール制約を必聞 |
| 経営者 | 「事業として／投資対効果」KPI 言及 | 価値判定（時間→金額）を厚く |
| クリエイター | 「発信／コンテンツ／ブランディング」 | 共有意図 = 不特定多数を疑う |

---

## 3. 文脈（context）

| 文脈 | シグナル |
|------|----------|
| 業務 | 顧客名／会議名／納期 |
| 個人 | 趣味／学習／生活効率 |
| 学習 | 「勉強したい／理解したい」 |
| 趣味 | 「面白そう／試したい」 |

文脈が業務の場合、**機密性配慮**が必須（クライアント名のマスキング等）。

---

## 4. 制約（constraints）

| 種類 | 例 |
|------|----|
| 時間 | 「夜しか触れない」「3 日後に必要」 |
| 予算 | 「無料で」「月額は厳しい」 |
| 権限 | 「会社の Google アカウントは触れない」 |
| 知識 | 「コードは書けない」 |

制約は `skill-intake-option-presenter` が選択肢から自動除外する。例: 予算ゼロ → 有料 API 候補を脱落させる。

---

## 5. 動機（motivation）

| 動機 | 「何が浮く？」への典型回答 |
|------|----------------------------|
| 効率化 | 時間／作業数 |
| 品質 | ミス率／クレーム |
| 学習 | 自分の理解度 |
| ブランディング | 露出／フォロワー |

動機は `value-realization-criteria.md` の KPI 設計に直結する。

---

## 6. 共有意図（share_target）

| 意図 | シグナル | 影響 |
|------|----------|------|
| 自分のみ | 「自分用に」 | プライバシー設計不要 |
| 少人数 | チーム名・固定メンバー | 共有相手リスト必聞 |
| 不特定多数 | SNS／フォーム公開 | 機密情報除外フィルタ必須 |
| 顧客 | クライアント名 | 機密性最高・監査ログ必要 |

---

## 推定の優先順位

1. 最初の発話 3 つで仮推定
2. `skill-intake-user-profiler` が 3 問追加で確定
3. 確定後は変更しない（フェーズ途中で揺れない）

## サンプル: google-forms-generator 想定ユーザー

```json
{
  "technical_level": "中級",
  "role": "個人事業主",
  "context": "業務",
  "constraints": ["時間=セミナー前で多忙", "知識=コードは書けない"],
  "motivation": "効率化",
  "share_target": "不特定多数"
}
```

このプロファイルを受け、後続 SubAgent は以下を自動適用する。

- 語彙: 中級辞書（API → 外部サービス連携、OAuth → Google にログイン許可 等）
- 図解: Mermaid 採用、独自 SVG も活用、★1 除外
- 選択肢: 無料サービス優先、設定画面で完結する手段を上位提示
