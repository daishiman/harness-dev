# 表層→深層変換辞書

ユーザーの最初の発話（表層）を真の課題（深層）に翻訳するパターン集。`skill-intake-assumption-challenger` と `skill-intake-purpose-excavator` 両 SubAgent がこの辞書を参照する。

## パターン一覧

| # | 表層 | よくある真の課題候補 |
|---|------|----------------------|
| 1 | 毎朝のスケジュールをまとめたい | 出力先未定／共有相手未定／通知タイミング／不要情報除外 |
| 2 | フォーム作りたい | 集計後の動作／通知先／期日制御／重複排除 |
| 3 | 議事録を自動化したい | 配信先／要点抽出粒度／参加者プライバシー |
| 4 | スライド作りたい | 聴衆／時間制約／結論優先度 |
| 5 | 論文をキャッチアップしたい | 取捨選択基準／配信頻度／読む時間の確保 |
| 6 | チラシ作りたい | 一次目的／媒体／訴求軸 |
| 7 | 契約書つくりたい | 雛形管理／レビュー者／改訂履歴 |
| 8 | プロンプト作りたい | 評価基準／反復方法／成果物の形 |
| 9 | X 投稿を自動化したい | トーン／頻度／ブランド整合 |
| 10 | リリース情報を集めたい | 取捨選択／更新検知／配信形態 |
| 11 | クライアント管理したい | プロファイル粒度／相互参照／機密扱い |
| 12 | 自分のメモを整理したい | 検索性／タグ統一／放置メモの再活性化 |

---

## パターン 1: 毎朝のスケジュールをまとめたい

| 観点 | 質問 | サンプル深層 |
|------|------|--------------|
| 出力先 | 「まとまったものはどこで見たい？」 | Slack DM／LINE／メール |
| 共有相手 | 「自分だけ？秘書？家族？」 | 自分のみ（プライバシー高） |
| 通知タイミング | 「何時に届けば嬉しい？」 | 朝 7 時／前夜 23 時 |
| 不要情報除外 | 「省きたい予定は？」 | 移動時間／繰返予定 |

**深層例**: 「毎朝 7 時に LINE で当日の重要予定 3 件のみ届く。移動・繰返は除外」

## パターン 2: フォーム作りたい

| 観点 | 質問 | 深層 |
|------|------|------|
| 集計後動作 | 「申込が来たら何が起きてほしい？」 | 自動メール返信／Slack 通知 |
| 期日制御 | 「いつまで受付？」 | セミナー前日 23:59 自動締切 |
| 重複排除 | 「同じ人の重複を許す？」 | 許さない（メアド一意） |

## パターン 3: 議事録を自動化したい

| 観点 | 質問 |
|------|------|
| 配信先 | Discord／メール／Notion |
| 要点抽出粒度 | 全文／決定事項のみ／アクションアイテム |
| プライバシー | 参加者名は伏字？／録音の保存期間 |

## パターン 4: スライド作りたい

| 観点 | 質問 |
|------|------|
| 聴衆 | 経営者／エンジニア／一般 |
| 時間制約 | 5 分／15 分／60 分 |
| 結論優先度 | 結論ファースト／物語型 |

## パターン 5: 論文をキャッチアップしたい

| 観点 | 質問 |
|------|------|

## パターン 6: チラシ作りたい

| 観点 | 質問 |
|------|------|
| 一次目的 | 集客／認知／ブランディング |

## パターン 7: 契約書つくりたい

| 観点 | 質問 |
|------|------|
| 雛形管理 | 単一マスター／複数バージョン |
| レビュー者 | 自分のみ／法務／顧問弁護士 |

| 観点 | 質問 |

| 観点 | 質問 |
|------|------|

|------|------|
| 検索性 | タグ／全文／日付 |

## 変換アルゴリズム

```
1. 表層要望 → パターン辞書で類似度マッチ
2. マッチしたパターンの「観点リスト」を取得
3. 各観点について standard 質問を 1 つずつ
4. 回答を集約 → 深層の一文に整形
5. Reverse Brief で確認
```

## サンプル: google-forms-generator

| 観点 | 質問 | ユーザー回答 |
|------|------|--------------|
| 集計後動作 | 「申込が来たら？」 | Sheets に自動追記。当日朝に確認したい |
| 期日制御 | 「締切は？」 | セミナー前日 23:59 |
| 重複排除 | 「重複は？」 | メアドで一意 |
| 出力先 | 「フォーム URL はどこに？」 | クリップボードと Slack DM |
| 共有相手 | 「誰が回答？」 | セミナー参加希望者（不特定多数） |
| 真の課題 | 「何が浮く？」 | 告知準備の手作業 90 分→10 分 |

→ 深層: 「セミナー告知 3 日前に 5 分でフォーム設置・自動集計、手作業 90 分削減」
