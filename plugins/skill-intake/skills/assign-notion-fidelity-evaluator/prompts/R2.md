## Layer 1: 基本定義層 (不変原則)

### 1.1 不変ルール
- Notion API を一切叩かず、JSON 構造比較のみで判定する。
- 12 section 全件で `granularity_score` を算出する (途中 break 禁止)。
- verdict=fail 時は即 exit 2 (fail-fast)、後続フェーズに進ませない。

### 1.2 倫理ガード
- 入力 JSON 内の PII フィールドはレポートに転記しない (key のみ列挙)。

## Layer 2: ドメイン層 (本質ロジック)

### 2.1 責務 (Single Responsibility)
- 担当: section 単位の char / field / viz 比較、加重平均と verdict 判定、JSON レポート出力。
- 非担当: Notion API 呼び出し、intake 内容の意味判定 (aggregator 責務)、Markdown 整形 (R3 責務)、canonical 派生 (R1 責務)。

### 2.2 ドメインルール

| 用語 | 定義 |
|---|---|
| `granularity_score` | section 単位の粒度一致度。素点 `0.30*char_score + 0.40*field_score + 0.30*viz_score` (0.0-1.0) を report では 100 倍し 0-100 スケールで出力 (欠落 section は `0`)。 |
| `char_score` | 本文文字数が canonical `[min,max]` 内なら 1.0、外なら境界からの乖離率で線形減衰。 |
| `field_score` | canonical `required_fields[]` のうち context に存在し型整合する比率。 |
| `viz_score` | canonical `viz_slots[mandatory=true]` のうち context.figures に対応 slot が存在する比率。 |
| `verdict` | `overall_score_ratio` を閾値判定: `pass(≥0.85) / warn(≥0.70) / fail(<0.70)`。`forced_fail=true` は無条件 fail。 |
| `overall_score` | 12 section 素点の加重平均を 100 倍した 0-100 スケール値。判定用に生比率 `overall_score_ratio` (0.0-1.0) も併記。加重は `absence_behavior=warn-fallback` の section が 0.5、それ以外 1.0。 |

ビジネスルール:
- `absence_behavior=block` の section が完全欠落 → 即 verdict=fail。
- `absence_behavior=warn` の section が欠落 → granularity_score=0.0 として算入。
- 閾値 (`fidelity_threshold_pass`, `fidelity_threshold_warn`) は CLI 引数で上書き可能。

### 2.3 入力契約

| field | type | required | 説明 |
|---|---|---|---|
| context-json (positional) | path | yes | `intake-final-context.json` |
| --pass-threshold | float | no | 既定 0.85 |
| --warn-threshold | float | no | 既定 0.70 |

### 2.4 出力契約
- schema: `schemas/output.schema.json` (additionalProperties:false)。真実源は `scripts/validate-notion-fidelity.py` の実出力 (`evaluate()` + `main()`)。
- 必須トップレベル: `sections[]`, `overall_score`, `overall_score_ratio`, `forced_fail`, `verdict`, `thresholds{pass,warn}`
- `sections[]` 各要素: `section_key`, `context_key`, `present`, `granularity_score`, `char_score`, `field_score`, `viz_score`, `missing_slots`, `excess_slots`, `missing_fields`, `warnings` (present=true 時のみ `text_length` を追加)

## Layer 4: 共通ポリシー層

### 4.1 失敗時挙動
- exit: 0=pass / 1=warn / 2=fail / 64=usage エラー (閾値順不正・context/snapshot 不在)。実体 `validate-notion-fidelity.py` に exit 3 経路は無い。
- context / snapshot ファイル不在・閾値順不正 → exit 64、stderr に理由、レポート未生成。
- 入力 JSON parse 失敗 → 未捕捉例外で異常終了 (クリーンな exit code は返さない)。
- E1 (12 section 完走) / E4 (block 欠落→forced_fail) は script が構造的に常時保証し、自己検査違反での中断経路は持たない。

### 4.2 観測 / ロギング
- verdict=fail → `fidelity-report.{json,md}` を out_dir (context.json と同階層) に書き出し stderr に verdict=fail を出力、exit 2 で公開を停止し skill-intake-handoff へ差し戻し。
- verdict=warn → 公開許可、`warnings[]` を CI ログへ伝播。

## Layer 5: エージェント層 (ゴール駆動の実行主体)

### 5.1 担当 agent
- `@fidelity-checker` (決定論バッチ、対話禁止、context-fork 不要)
- 実体: `scripts/validate-notion-fidelity.py`

### 5.2 ゴール定義
- 目的: intake-final-context と canonical snapshot を構造比較し、section 単位の granularity_score と overall verdict を決定論的に算出する。
- 背景: Notion API 経由検証は不安定で再現性が低い。JSON 構造比較で公開可否判定を機構化し fail-fast を保証する必要がある。
- 達成ゴール: fidelity-report.json (schema 準拠) が 12 section 完備で生成され、verdict と exit code が 1:1 に確定し、blocker 違反時は fail で即停止する状態。

### 5.3 完了チェックリスト (ゴール到達の停止条件)
- [ ] E1: `len(report.sections) == 12` (途中 break なし)
- [ ] E2: `report.verdict in {pass, warn, fail}`
- [ ] E3: exit code が verdict と 1:1 (`pass=0 / warn=1 / fail=2`、usage エラー=64)
- [ ] E4: `report.forced_fail == true` (absence_behavior=block の section 欠落) のとき verdict は必ず `fail`
- [ ] E5: 同一 (context, canonical) で連続 2 回実行し report が bit-identical (determinism)
- [ ] E6: 全 section の `char_score/field_score/viz_score ∈ [0.0, 1.0]` かつ `granularity_score ∈ [0.0, 100.0]`
- [ ] E7: Notion API 等の外部 HTTP を一切起動していない
- [ ] E8: PII フィールド値が report に転記されていない (key のみ列挙)

### 5.4 実行方式
- 固定手順を持たない。未充足チェック項目を特定→解消手順を都度立案 (snapshot ロード / score 算出 / blocker 検査 / atomic write など)→実行→チェックリストで自己評価→全項目充足まで反復 (上限: Layer 4 最大反復回数)。
- 逸脱時: 入力不正 (閾値順不正・context/snapshot 不在) は Layer 4.1 の exit 64 で停止する (レポート未生成)。

## 出力指示 (LLM 実行時に読む箇所)

LLM はここから下の指示のみを実行し、Layer 1〜7 はコンテキストとして参照する。

`{{context_json}}` を読み込み、`references/canonical-page-snapshot.json` と突合せよ。各 section の char/field/viz スコアを算出し、`granularity_score = (0.30*char + 0.40*field + 0.30*viz) * 100` (0-100 スケール) で集約。`absence_behavior=block` の section 欠落は `forced_fail=true` とし即 verdict=fail。`overall_score_ratio` の加重平均から `thresholds` で verdict を確定し、`fidelity-report.json` を atomic write、verdict に応じた exit code を返すこと。前置き禁止。
