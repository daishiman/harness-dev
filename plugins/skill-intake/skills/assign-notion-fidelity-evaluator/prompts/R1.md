## Layer 1: 基本定義層 (不変原則)

### 1.1 不変ルール
- snapshot は SoT (`section_canonical_map.json`) から決定論的に派生する。手書き禁止。
- snapshot 更新は `audit_trigger=template-change` 発火時のみ。
- 12 section (§0〜§11) を欠落なく含める。
- 出力先の親ディレクトリを作成し、snapshot 全文を一括 `write_text` で書き出す (部分派生を残さない)。

### 1.2 倫理ガード
- Notion API トークンを扱わない (静的派生のみ、資格情報を環境変数から読まないこと)。

## Layer 2: ドメイン層 (本質ロジック)

### 2.1 責務 (Single Responsibility)
- 担当: SoT 読込 / 構造抽出 / 派生メタ付与 / snapshot 書き出し。
- 非担当: Notion API 呼び出し、内容検証 (R2 責務)、Markdown 整形 (R3 責務)。

### 2.2 ドメインルール

| 用語 | 定義 |
|---|---|
| `canonical-page-snapshot` | info-collector-agent Notion ページ構造を section / required_fields / char_bounds / viz_slots に分解した JSON。本スキル内 fidelity 比較の正本。 |
| `section_canonical_map` | `plugins/skill-intake/references/section_canonical_map.json` (v2)。snapshot の唯一の派生元 (SoT)。 |
| `audit_trigger=template-change` | SoT スキーマ変更検知時のフラグ。R1 が再実行される唯一の条件。 |

成功基準: `snapshot.sections.length == 12` かつ `snapshot.derived_from.sha256` が SoT の sha256 と一致。

### 2.3 入力契約

| field | type | required | 説明 |
|---|---|---|---|
| --source | path | yes | SoT JSON (`section_canonical_map.json`) |
| --out | path | yes | snapshot JSON 出力先 |

### 2.4 出力契約
- schema: `references/canonical-page-snapshot.json` (本スキル直下、派生キャッシュ)
- 必須フィールド (トップレベル): `schema_version`, `generated_at`, `canonical_page_id`, `derived_from{path, sha256}`, `sections[]`
- `sections[]` の各要素: `section_key`, `user_canonical_section`, `title`, `required_fields[{key, type, absence_behavior}]`, `char_bounds{min, max}`, `viz_slots[{role, asset_id, mandatory}]`, `absence_behavior`

## Layer 3: インフラ層 (外部依存)

### 3.1 参照リソース

| id | path | when_to_read |
|---|---|---|
| SoT | plugins/skill-intake/references/section_canonical_map.json | 派生実行時 (--source) |
| 抽象化契約 | plugins/skill-intake/skills/assign-notion-fidelity-evaluator/references/abstraction-contract.md | テンプレ変数差し替え時 |

### 3.2 外部ツール / API
- Python 3 (argparse, datetime, hashlib, json, sys, pathlib)
- 外部 HTTP 禁止 (Notion API 含む)

## Layer 4: 共通ポリシー層

### 4.1 失敗時挙動
- SoT 不在 → exit 64 (usage error)、stderr に `[extract-canonical-snapshot] source not found: <path>`。
- SoT JSON parse 失敗 → 未捕捉例外で exit 1 (Python traceback を stderr)。
- 正常派生 → exit 0。

### 4.2 観測 / ロギング
- stdout に `[extract-canonical-snapshot] wrote <out> (<N> sections)`。
- `snapshot.generated_at` は ISO 8601 + JST offset (`+09:00`)。

### 4.3 セキュリティ
- secret / token を扱わない。資格情報を環境変数から読まないこと。

## Layer 5: エージェント層 (ゴール駆動の実行主体)

### 5.1 担当 agent
- `@canonical-snapshot-extractor` (非対話バッチ、context-fork 不要)
- 実体: `scripts/extract-canonical-snapshot.py`

### 5.2 ゴール定義
- 目的: SoT (`section_canonical_map.json`) から `canonical-page-snapshot.json` を決定論的に派生し fidelity 比較の正本キャッシュを与える。
- 背景: 手書き snapshot は SoT との乖離を生み R2 の判定根拠を腐らせる。派生機構と一括書き出しを機構で固定する必要がある。
- 達成ゴール: snapshot が 12 section 完備かつ SoT の sha256 と derived_from.sha256 が一致し、`write_text` で書き出し済みの状態。

### 5.3 完了チェックリスト (ゴール到達の停止条件)
- [ ] E1: `len(snapshot.sections) == 12` (欠落ゼロ、`section_key` の §0〜§11 順を保持)
- [ ] E2: `snapshot.derived_from.sha256 == sha256(open(source).read())` (並行更新検知)
- [ ] E3: `snapshot.schema_version == source.schema_version` (継承一貫性)
- [ ] E4: 出力先の親 dir 作成後に snapshot 全文を `write_text` で書き出している (部分派生なし)
- [ ] E5: 同一 source で連続 2 回実行し snapshot が bit-identical (determinism)
- [ ] E6: Notion API 等の外部 HTTP / secret 読込を行っていない (倫理ガード)
- [ ] E7: `snapshot.generated_at` が ISO 8601 + JST offset (`+09:00`) 形式

### 5.4 実行方式
- 固定手順を持たない。未充足チェック項目を特定→解消手順を都度立案 (SoT parse / section 抽出 / sha256 算出 / snapshot 書き出し など)→実行→チェックリストで自己評価→全項目充足まで反復 (上限: Layer 4 最大反復回数)。
- 逸脱時: SoT 不在は Layer 4.1 の exit code 規約 (exit 64)、SoT parse 失敗は未捕捉例外 (exit 1) で停止。

## Layer 6: オーケストレーション層

### 6.1 上位 skill との接続
- 呼び出し元: `assign-notion-fidelity-evaluator` SKILL.md Step 1 (template-change trigger 時のみ)

## 出力指示 (LLM 実行時に読む箇所)

LLM はここから下の指示のみを実行し、Layer 1〜7 はコンテキストとして参照する。

`{{source}}` (SoT) を読み込み、12 section を `section_key` の §0〜§11 順で抽出し、SoT 全文 sha256 を持つ `derived_from{path, sha256}` と、トップレベルの `generated_at` (ISO8601+09:00) / `canonical_page_id` を付与せよ。snapshot 全文を `{{out}}` の親 dir 作成後に `write_text` で書き出し、stdout に `[extract-canonical-snapshot] wrote <out> (<N> sections)` を出力。失敗時は Layer 4.1 の exit code 規約 (SoT 不在=exit 64 / parse 失敗=exit 1 / 正常=exit 0) に従うこと。前置き禁止。
