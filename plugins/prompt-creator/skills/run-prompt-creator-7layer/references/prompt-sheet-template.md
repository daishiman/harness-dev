# Prompt作成シート テンプレート

ヒアリング結果を埋めるためのテンプレート。

## テンプレート

```markdown
# Prompt作成シート

# Next Action
- [ ] フォーマットを具体的な内容で埋める

# プロンプト名
> 素人でもこれを見ただけで分かるこのプロンプトを命名
- {{prompt_name}}

# プロンプトの想定利用者
> このプロンプトを実際に使う人物像や役割
- {{target_user}}

# プロンプト作成の目的
> このプロンプトを作ることで達成したい最終ゴール
- {{purpose}}

# プロンプト作成の背景
> なぜこのプロンプトが必要になったのか、その経緯や状況
- {{background}}

# 完了条件（成功基準）
> どのような状態になれば「このプロンプトは完成した」と判断できるか
- {{success_criteria}}

# 達成ゴール（成果状態）
> 何が出来上がれば到達か。手順ではなく成果状態で記述する。手順は実行時にAIが自律生成する。
1. {{goal_1}}
2. {{goal_2}}

# 完了チェックリスト（ゴール到達の停止条件）
> 第三者がYES/NOで判定できる達成条件。これが満たされるまでAIはゴールに向け反復する。
- [ ] {{checklist_1}}
- [ ] {{checklist_2}}

# 成果物フォーマット
> ゴール達成時に出力してほしい形式

\```
{{output_format}}
\```

# 解決すべき課題
> プロンプトが解消するべき問題点や現状の不便
- {{challenge_1}}
- {{challenge_2}}
- {{challenge_3}}

# 目的を達成するために必要な情報
> プロンプトが高品質な出力を行うために、事前に準備・入力しておくべき具体的な情報。
- {{required_info_1}}
- {{required_info_2}}
- {{required_info_3}}

# 注意点・制約条件
> プロンプト設計時に守るべき条件や避けたい出力
- ゴール未達の間はチェックリストを基に手順を都度生成して反復する
- {{constraint_1}}
- {{constraint_2}}

# プロンプトに対する課題
- {{prompt_issue}}

# 想定するユーザー入力1
\```
{{test_input_1}}
\```

# 期待される出力1
\```
{{test_output_1}}
\```

# 想定するユーザー入力2
\```
{{test_input_2}}
\```

# 期待される出力2
\```
{{test_output_2}}
\```

# 想定するユーザー入力3
\```
{{test_input_3}}
\```

# 期待される出力3
\```
{{test_output_3}}
\```

# 想定するユーザー入力4
\```
{{test_input_4}}
\```

# 期待される出力4
\```
{{test_output_4}}
\```

# 想定するユーザー入力5
\```
{{test_input_5}}
\```

# 期待される出力5
\```
{{test_output_5}}
\```
```

## 埋め方ガイド

| フィールド | 情報源 | 優先度 |
|-----------|--------|-------|
| プロンプト名 | ヒアリングQ1から命名 | 必須 |
| 想定利用者 | ヒアリングQ2 | 必須 |
| 目的 | ヒアリングQ1 | 必須 |
| 背景 | ヒアリングQ3 | 必須 |
| 完了条件 | ヒアリングQ7 or Q1から推定 | 必須 |
| 達成ゴール | ヒアリングQ5（ゴール・完了条件） | 必須 |
| 完了チェックリスト | ヒアリングQ5から導出 | 推奨 |
| 成果物フォーマット | ヒアリングQ4 | 必須 |
| 課題 | ヒアリングQ3 | 必須 |
| 必要情報 | ゴールから逆算 | 推奨 |
| 制約条件 | ヒアリングQ6 | 推奨 |
| テストケース | ヒアリングQ8 | オプション |
