# 記述スタイル原則

prompt-creator が生成するすべてのプロンプトに適用する不変原則。

---

## 原則1: ルールは「本体 + 目的 + 背景」をセットで記述する

- **目的**: 読み手がエッジケースで適切に判断できるようにするため。
- **背景**: 目的なきルールは形骸化しやすく、状況に応じた応用判断もできない。

**フォーマット**:
```
- **CONST_NNN (ルール名)**: <ルール本体を1〜2文>
  - **目的**: <防ぐ事象・実現したい状態>
  - **背景**: <採用の根拠・過去の失敗事例・制約>
```

---

## 原則2: 大まかな流れで書く（細部はAIに委ねる）

- **目的**: 重要な目的・背景・制約に注意を集中させ、トークンを浪費しないため。
- **背景**: 細部手順を冗長列挙しても LLM は全文を精密に解釈せず、遵守率も下がる。

**書く**: 各ステップの目的・完了条件・禁止事項・ユーザー確認点
**書かない**: 細かな分岐の網羅列挙、自明な常識的処理の手順

---

## 原則3: 1文で伝わるものを段落で書かない

- **目的**: 読まれない長文より、必ず読まれる短文を優先するため。
- **背景**: 長大なプロンプトは LLM もユーザーも全文精読しない。

---

## 原則4: 出力フォーマットは Markdown を既定とする

- **目的**: 人間がレビュー・編集しやすくするため。
- **背景**: JSON/YAML 単体は機械可読だが、人間が編集する本体には不向き。

---

## レビューチェック

- [ ] 全 CONST_NNN / 主要ルールに「目的」「背景」併記
- [ ] 細部手順を冗長列挙していない
- [ ] 同一内容を別言い回しで重複説明していない
- [ ] Markdown 形式
