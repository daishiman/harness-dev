# 冪等更新ポリシー（重複回避・上書き優先 + セッション分離）

ゴールシークでプロンプトを改善・更新するときの正本。review-prompt / generate-prompt
両エージェントと quality-criteria.md が参照する。狙いは **再現性高く品質を上げる** こと。

## 1. 位置づけ（上位規範と用途境界）

本ポリシーは「重複→上書き一本化」規範の **prompt-creator 改善ループ適用版**。鉄則と理由
（keep-both 禁止 / 二重定義は片方更新で必ず矛盾し判断コストを後続へ転嫁）は上位正本
`plugins/harness-creator/references/ssot-dedup-procedure.md` (ANTI-DUP-001) を参照し再掲しない。

用途境界（両者は用途非互換のため統合せず分離する。境界は上位規範の正本決定規則に準拠）:

| 正本 | 担当する重複 | 検出手段 |
|---|---|---|
| ssot-dedup-procedure.md (上位) | プラグイン横断の構造的 SSOT — schema `$id` / redirect 本文 / 6 行 passage / ファイル単位の正本 | `lint-ssot-duplication.py`（機械） |
| 本ポリシー（下位適用版） | 単一プロンプト内のゴールシーク改善時の意味的・要素レベル重複（1 ゴール / 1 ルール / 1 責務 / 1 チェック項目）+ セッション分離 | review-prompt の質的判定（LLM） |

- **目的**: 改善のたびに闇雲に追加せず、情報肥大化と「どちらを採用すべきか」の曖昧化を防ぐ（再現性高く品質を上げる）。要素原子性（quality-criteria §5）とも整合させる。

## 2. 冪等更新アルゴリズム（ゴールシーク反復の各サイクルで実行）

```
1. 既存プロンプトを原子要素へ分解
     原子要素 = Layer 内の 1 ゴール / 1 ルール / 1 エージェント責務 / 1 チェックリスト項目
2. 提案改善ごとに、既存要素との「類似」を質的判定（§3）
3. 類似要素あり  → その最類似要素を上書き更新（新規追加しない）
4. 類似要素なし  → 新規追加
5. 完了判定      → 同一意図の要素が 2 つ以上存在しない（重複ゼロ）
```

最初に **既存内容の分析→分解** を必ず行う。分析前に追加・上書きを決めてはならない。

## 3. 類似判定基準（数量でなく意図で）

次のいずれかが YES なら「類似」とみなし、新規追加でなく **上書き対象** とする。

| 観点 | 判定質問 |
|---|---|
| 同一対象 | 同じ Layer / 同じエージェント / 同じ Tool / 同じ完了条件を指すか |
| 同一意図 | 達成したい成果状態が同じか |
| 同一改善ポイント | 解こうとしている問題（矛盾・漏れ・冗長など）が同じか |

3 観点すべて NO のときのみ「新規」と判定する。

## 4. 上書き時の品質保持

- 上書きは **情報を失わない統合**：両者の優れた点をマージし、劣る表現のみ破棄する。
- 上書き後も元の検証可能性（第三者が YES/NO 判定できる）を維持する。
- マージで長文化したら要素原子性（1 値 50 文字目安）に従い再分解する。
- 削除でなく統合のため、上書き前後を trace に差分記録し再現性を担保する。

## 5. セッション分離（ゴールシーク起動時は必須）

ゴールシーク反復は **SubAgent / エージェントチームで分離 context** で実行する。

- **目的**: 反復中の中間情報（分解結果・類似判定・差分候補・没案）が現セッションを汚染するのを防ぐ。
- **背景**: ゴールシークは多くの探索情報を生むため、親 context に混ぜると後続判断が劣化する。
- **規約**:
  - 親 context は SubAgent へ「対象パスとゴール+完了チェックリスト」のみ渡す。
  - SubAgent は反復・探索を内部で完結させ、親へは **最終差分と完了判定のみ** を返す。
  - 中継は `shared_state`（要約 200 字以内）に限定し、中間探索ログを親へ流さない。
  - 分離先は役割（担当 SubAgent）単位に分け、各役割の入力・責務・出力・編集範囲・依存関係を明示する。同じ判断責務を複数の役割へ重複させない。

elegant-review の Phase 1 リセット（context fork 強制）/ Phase 2 並列分析（独立 SubAgent）/
`shared_state.md` 200 字制限と同型。整合させること。

## 6. 完了チェックリスト（このポリシーの達成条件）

- [ ] 改善前に既存内容を原子要素へ分解・分析した
- [ ] 各提案を §3 で類似判定し、上書き / 新規を一意に決定した
- [ ] 同一意図の要素が 2 つ以上存在しない（重複ゼロ）
- [ ] 上書きは情報を失わない統合で、検証可能性を維持した
- [ ] ゴールシーク反復を分離 context（SubAgent/チーム）で実行し、親へは最終差分のみ返した
