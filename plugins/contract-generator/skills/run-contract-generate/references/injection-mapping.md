# 差込マッピング(人間向け要約)

機械可読 SSOT は `template-mapping.json`。本書はその要約と背景。台帳列⇄ひな形プレースホルダの対応を示す。

## 甲 (固定値) の SSOT

甲側 (サンプル株式会社 等) は台帳列にせず、`lib/config_auth.load_party_a()` (= `lib/ledger.get_party_a()`) を SSOT とする。差込時は `template-mapping.json` の `common.fixed_values` に置かれた `{{party_a.name}}` `{{party_a.address}}` `{{party_a.representative}}` `{{party_a.title}}` `{{party_a.rep_name}}` を解決する。フォールバック優先順位の正本は `../../../references/party_a-readme.md`(複製しない)。


## 個人(個人②ひな形 / 台帳「個人」シート)

| 台帳列 | ひな形位置(anchor) | プレースホルダ | 種別 |

| 乙氏名・名称 | 冒頭「（以下「乙」」/ 署名欄「乙：」 | ● | 直接(2箇所) |

## 法人(法人①ひな形 / 台帳「法人」シート)

個人との差分のみ:
- 乙列が **乙法人名・名称 / 乙代表者役職・氏名 / 乙本店所在地・連絡先** の3列。署名欄に代表者を差込む。
- **遂行場所・成果物番号・納品期限・検収期間が無い**(法人①は本体に該当条項なし=別紙1委譲)。
- 第7条フリーランス法明示義務は条文側固定(差込なし)。

## 条件分岐(置換でなく文面の有無切替)

| 列 | 値 | 動作 |
|---|---|---|
| 業務内容方式 | する / 別紙1に定める | ［する／…別紙１…］ の選択 |
| 料金方式 | 固定金額 / 別紙2参照 | ［金XXXX円／別紙２…］ の選択 |
| 個人情報処分方法 | 別途定める / 別紙3に定める | ［甲が別途定める／別紙３…］ の選択 |
| 成果物有無(個人のみ) | あり / なし | なし=第2条3・5項を段落削除 |
