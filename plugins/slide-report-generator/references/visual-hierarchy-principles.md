# 視覚階層の設計原則

> **正本**: [spec-registry.md](spec-registry.md) — このファイルは設計の文脈・例・適用ガイドのみ。規則の正本は SR-ID で参照すること

**責務**: スライド上の情報に優先度をつけ、聴衆の視線を意図的に誘導するための設計原則。
**規則の正本**: フォントスケール → [SR-3-03](spec-registry.md#sr-3-03)、ビビッドアクセント上限 → [SR-2-06](spec-registry.md#sr-2-06)、stagger/easing → §6

---

## 1. 視覚階層とは

聴衆が「何を最初に見て、次にどこを見て、最後に何を読むか」をデザインで制御する技術。1スライドに3段階以内の情報階層を作る。

### 3段階モデル

| レベル | 役割 | 認知時間 | デザイン手法 |
|--------|------|---------|------------|
| L1: ヒーロー | 最初に目に入る要素 | 0.5秒以内 | 最大サイズ、ビビッドカラー、中央配置 |
| L2: サポート | L1を補足・説明 | 1-2秒 | 中サイズ、アクセントカラー、L1近傍 |
| L3: ディテール | 補足情報・注釈 | 3秒以降 | 小サイズ、ミュートカラー、端配置 |

### アンチパターン

```
全要素が同じサイズ・同じウェイト → 階層なし → 何を見ればよいか不明
```

### 適用例

```
[ヒーロー数値: 4.5x 大きく太く青]
  ↓ 視線移動
[説明文: 中サイズ、通常ウェイト]
  ↓ 視線移動
[注釈: 小サイズ、グレー]
```

## 2. 階層構築の5つの手段

### 2.1 サイズ差（Scale）

最も直感的な手段。**L1とL3のサイズ差が2倍未満なら階層が弱すぎる**（検証 S14）。

| 階層 | フォントサイズ変数 | 倍率目安 |
|------|------------------|---------|
| L1 | --fs-title / --fs-display | 3.0x-4.0x |
| L2 | --fs-heading / --fs-subheading | 1.5x-2.0x |
| L3 | --fs-body / --fs-small | 1.0x（基準） |

各変数の正値 → [SR-3-03](spec-registry.md#sr-3-03)、最小サイズ → [SR-3-04](spec-registry.md#sr-3-04)

### 2.2 ウェイト差（Weight）

同一サイズでも太さで階層を作れる。**1スライド内でウェイトは最大3種類まで**（4種以上は混乱）。

| 階層 | ウェイト | 用途 |
|------|---------|------|
| L1 | --fw-bold (700) | タイトル、ヒーロー数値 |
| L2 | --fw-semibold (600) | 見出し、キーワード |
| L3 | --fw-regular (400) | 本文、説明 |

### 2.3 色差（Color）

カラー戦略は [color-strategy.md](color-strategy.md) を参照。アクセント色の上限規則 → [SR-2-06](spec-registry.md#sr-2-06)。

| 階層 | カラー戦略 |
|------|-----------|
| L1 | --accent-*-vivid |
| L2 | --wave-blue / --sakura-pink 等 |
| L3 | --fg-dim / --fg-muted |

### 2.4 コントラスト差（Contrast）

WCAG コントラスト規則 → [SR-9-01](spec-registry.md#sr-9-01)。

```css
.hero-text { color: var(--accent-blue-vivid); }   /* 高コントラスト L1 */
.caption-text { color: var(--fg-muted); }         /* 低コントラスト L3 */
```

### 2.5 空間配置（Position）

人間の視線は特定パターンで移動する。重要な要素はそのパターンの**起点**に配置する。

| パターン | 視線の流れ | 適用スライド |
|---------|-----------|------------|
| F型 | 左上→右→左下→右 | テキスト多めのスライド |
| Z型 | 左上→右上→左下→右下 | 画像+テキスト混在 |
| 中央集中 | 中央→外周 | ヒーロー数値、引用 |
| 上→下 | 上部→下部 | フロー、タイムライン |

## 3. スライドタイプ別ヒエラルキーパターン（適用例）

### 3.1 タイトルスライド

```
L1: タイトル（--fs-title, bold, 中央）
L2: サブタイトル（--fs-subtitle, regular, タイトル直下）
L3: 発表者名・日付（--fs-small, --fg-dim, 最下部）
```

**設計の意図**: タイトルとサブタイトルの間に十分な空白（--space-6以上）。近すぎると窮屈。

### 3.2 キーメッセージスライド

```
L1: キーフレーズ（--fs-heading, bold, --accent-blue-vivid, 中央）
L2: 補足文（--fs-body, regular）
L3: なし（余白を活かす）
```

**設計の意図**: 要素を**減らす**ことが最大の強調。Apple Keynote では1行+余白80%がヒーロー。

### 3.3 統計・数値スライド

```
L1: 数値（--fs-display, bold, --accent-*-vivid）
L2: 単位・ラベル（--fs-subheading, medium, 数値近傍）
L3: 比較・出典（--fs-small, --fg-dim, 下部）
```

**設計の意図**: 数値は**色と位置**で孤立させる。周囲の空白で数値を浮き立たせる。

### 3.4 リスト・カードスライド

```
L1: セクションタイトル（--fs-heading, semibold, 上部）
L2: カード見出し（--fs-subheading, semibold, カード内上部）
L3: カード本文（--fs-body, regular）
```

**設計の意図**: カード間 gap > カード内 padding にしてグループ化を明示。

### 3.5 比較スライド（Before/After、vs）

レイアウト比率規則 → [SR-4-03](spec-registry.md#sr-4-03)、色対応 → [SR-4-04](spec-registry.md#sr-4-04)

```
L1: 比較軸ラベル（--fs-subheading, bold, 各パネル上部）
L2: 比較内容（--fs-body, 各パネル内）
L3: 結論・矢印（--fs-body, --accent-aqua-vivid, 中央 or 下部）
```

**設計の意図**: 左右パネルの色調を変えて対比を視覚化。

### 3.6 フロー・プロセススライド

```
L1: フロー全体のタイトル（--fs-heading, 上部）
L2: 各ステップラベル（--fs-subheading, ステップ内）
L3: ステップ詳細（--fs-small, ステップ内下部）
```

**設計の意図**: 矢印・コネクタは控えめな色（--fg-muted）。コネクタが主役になってはいけない。

## 4. フォーカルポイント設計

### 4.1 フォーカルポイントとは

聴衆の視線が**最初に固定される1点**。1スライドに**1つだけ**設定する。

### 4.2 フォーカルポイントの作り方

| 手法 | 実装 | 効果 |
|------|------|------|
| サイズ突出 | 1要素だけ2倍以上大きく | 最も確実 |
| カラー孤立 | 1要素だけビビッドカラー | 色のコントラストで注目 |
| 空白囲い | 周囲に十分な余白 | 孤立感 |
| 動き | enterアニメーション最初 | 動的注目 |
| 位置特権 | 中央 or 左上 | 自然な視線誘導 |

### 4.3 アンチパターン

| NG | 理由 | 改善 |
|----|------|------|
| 全要素にビビッドカラー | 強調が消える | [SR-2-06](spec-registry.md#sr-2-06) に従う |
| 全要素同時アニメーション | フォーカルポイントなし | stagger で順序付け |
| テキストのみ・同サイズ | 階層なし | L1を最低2倍サイズに |
| 装飾過多 | ノイズ | 装飾は機能に従う |

## 5. 視線誘導のアニメーション設計

GSAP 仕様の正本 → §6（[SR-6-01](spec-registry.md#sr-6-01)〜[SR-6-08](spec-registry.md#sr-6-08)）

### 5.1 stagger による読み順制御

```javascript
gsap.timeline()
  .from('.hero-number', { opacity: 0, y: 20, duration: 0.35, ease: 'power2.out' })
  .from('.supporting-text', { opacity: 0, y: 15, duration: 0.25, ease: 'back.out(1.7)' }, '-=0.15')
  .from('.caption', { opacity: 0, duration: 0.2, ease: 'power1.inOut' }, '-=0.1');
```

### 5.2 方向性による視線誘導

| アニメーション方向 | 視線誘導先 | 用途 |
|------------------|-----------|------|
| 上→下（y: -20 → 0） | 下方向 | タイトル→本文 |
| 左→右（x: -20 → 0） | 右方向 | フロー・タイムライン |
| 拡大（scale: 0.8 → 1） | 中央 | ヒーロー要素（[SR-6-01](spec-registry.md#sr-6-01) 厳守） |
| フェードイン（opacity: 0 → 1） | その場 | 補足・ディテール |

### 5.3 階層別アニメーション強度

| 階層 | 移動距離 | duration | イージング |
|------|---------|----------|-----------|
| L1 | 20-30px | 0.3-0.5s | power2.out / back.out |
| L2 | 10-20px | 0.2-0.3s | power2.out |
| L3 | 0-10px | 0.15-0.2s | power1.inOut |

**設計の意図**: 階層が下がるほど動きを控えめに。L3が派手に動くとL1から視線を奪う。イージング種類規則 → [SR-6-05](spec-registry.md#sr-6-05)。

## 6. デザインセンス判定チェックリスト

### 必須チェック（規則）

- [ ] L1とL3のサイズ差 2倍以上 → S14 / [SR-3-03](spec-registry.md#sr-3-03)
- [ ] ウェイト種類 3種以内（運用基準）
- [ ] ビビッドカラー 2色以内 → [SR-2-06](spec-registry.md#sr-2-06)
- [ ] stagger 読み順 → [SR-6-05](spec-registry.md#sr-6-05) / [SR-6-06](spec-registry.md#sr-6-06)
- [ ] フォーカルポイント周囲の余白

### 品質チェック（運用判断）

- [ ] テキストのみのスライドにL1/L2の差があるか
- [ ] カードスライドで gap > padding か
- [ ] 比較スライドで左右の色調が異なるか
- [ ] フローのコネクタが主役より控えめな色か
