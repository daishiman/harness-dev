# 配色戦略・色彩設計原則

> **正本**: [spec-registry.md](spec-registry.md) — このファイルは設計の文脈・例・適用ガイドのみ。規則の正本は SR-ID で参照すること

**責務**: スライドの配色をどう「決める」かの判断基準。色彩心理・スライドタイプ別配色パターン・アンチパターンを提供。
**規則の正本**: §2 カラー（SR-2-01〜SR-2-09）

---

## 1. 配色の基本原則：60-30-10ルール

- 配分比率の規則 → [SR-2-06](spec-registry.md#sr-2-06)（1スライドあたりビビッドアクセント最大2色・10%以内）
- 各スライドにビビッドアクセント1色以上 → [SR-2-05](spec-registry.md#sr-2-05)

| 比率 | 役割 | Kanagawaテーマでの対応 |
|------|------|----------------------|
| 60% | ベース（背景+余白） | --bg-dark |
| 30% | セカンダリ（テキスト+パネル） | --fg + --bg-card |
| 10% | アクセント（強調点） | --accent-*-vivid |

**設計の意図**: アクセントは面積10%以下。超えると「強調」が「主色」に変わり、ベースが背景に見えなくなる（視覚的フラット化）。

## 2. カラーロール（色の役割定義）

カラー変数定義は [SR-2-02](spec-registry.md#sr-2-02) / [SR-2-03](spec-registry.md#sr-2-03) / [SR-2-04](spec-registry.md#sr-2-04) を参照。

### 2.1 5つのカラーロール

| ロール | 責務 | 変数 | 使用箇所 |
|--------|------|------|---------|
| Background | 背景・余白 | --bg-dark, --bg-dim | body, slide背景 |
| Surface | カード・パネル面 | --bg-card, --bg-highlight | カード, テーブル |
| Primary Text | 主要テキスト | --fg | 見出し, 本文 |
| Secondary Text | 補助テキスト | --fg-dim, --fg-muted | キャプション, 注釈 |
| Accent | 強調・CTA | --accent-*-vivid | キーワード, ボタン, 数値 |

### 2.2 ロール間のコントラスト

コントラスト比規則 → [SR-9-01](spec-registry.md#sr-9-01)

実測値（参考）:
- #43436c on #fafafa = 7.2:1 PASS
- #43436c on #e8e8ec = 5.1:1 PASS
- #888888 on #fafafa = 3.5:1（大テキストのみ）

## 3. 色彩心理マッピング

意味と色の対応の規則 → [SR-2-07](spec-registry.md#sr-2-07)

### 3.1 スライドの感情別カラー選択（運用ガイド）

| 感情・意図 | 推奨アクセント | 理由 |
|-----------|--------------|------|
| 信頼・安定 | --accent-blue-vivid | 青=信頼性・専門性 |
| 緊急・課題 | --accent-pink-vivid | 赤系=注意・緊急性 |
| 成長・解決 | --accent-aqua-vivid | 緑系=成功・解決・前進 |
| 創造・革新 | --accent-violet-vivid | 紫=創造性・高級感 |
| 注意・CTA | --accent-yellow-vivid | 黄=注意・行動喚起 |

### 3.2 プレゼン文脈別の配色戦略

| 文脈 | 主要アクセント | 補助アクセント | 理由 |
|------|--------------|--------------|------|
| 提案・営業 | blue | aqua | 信頼→解決の流れ |
| 課題提起 | pink | blue | 危機感→信頼で安心 |
| 成果報告 | aqua | yellow | 達成感+数値強調 |
| ブランディング | violet | pink | 高級感+差別化 |
| 教育・研修 | blue | violet | 専門性+知的好奇心 |

## 4. スライドタイプ別配色パターン（適用例）

### 4.1 タイトルスライド

```
背景: --bg-dark or --gradient-blue-pink
タイトル: --fg or 白（グラデ背景時）
サブタイトル: --fg-dim
アクセント: --accent-blue-vivid
```

**設計の意図**: タイトルスライドのみグラデーション背景を許容。本編で使うと特別感が薄れる。

### 4.2 キーメッセージ

```
背景: --bg-dark
テキスト: --accent-*-vivid（メッセージの感情に合わせて選択）
補足: --fg-dim
```

**設計の意図**: キーメッセージは1色に絞る。複数色は焦点を分散させる。

### 4.3 統計・数値

```
数値: --accent-*-vivid（最も主張したい感情色）
単位: --fg
比較元: --fg-muted
成長矢印: aqua（上向き）/ pink（下向き）
```

**設計の意図**: ポジティブ=aqua、ネガティブ=pink を全スライドで一貫させる（[SR-2-07](spec-registry.md#sr-2-07)）。

### 4.4 比較スライド

レイアウト・色対応の規則 → [SR-4-03](spec-registry.md#sr-4-03) / [SR-4-04](spec-registry.md#sr-4-04)

**設計の意図**: 左右で背景色を微妙に変えて対比を視覚化。テキスト色は統一して可読性を維持。

### 4.5 フロー・プロセス

```
ステップ: --bg-card背景、--fg テキスト
コネクタ/矢印: --fg-muted（控えめ）
現在ステップ: --accent-blue-vivid ボーダー
完了ステップ: --accent-aqua-vivid アイコン
```

**設計の意図**: フロー全体を同一色調にし、「現在地」だけアクセントで際立たせる。

### 4.6 カード・グリッド

```
カード背景: --bg-card or .glass-card
カードタイトル: --fg + semibold
カード本文: --fg-dim
アイコン: 各カードのテーマに合った --accent-*-vivid（1種類/カード）
```

**設計の意図**: アイコン色を統一すれば「結束感」、変えれば「分類感」。目的に応じて選択。

## 5. カラーのアンチパターン

### 5.1 絶対禁止

| NG | 理由 | 根拠 SR-ID |
|----|------|-----------|
| 純黒 (#000000) テキスト | コントラスト高すぎ・目が疲れる | — |
| 純白 (#FFFFFF) 背景 | Lotus は #fafafa | [SR-2-01](spec-registry.md#sr-2-01) |
| ネオンカラー | Kanagawa から逸脱 | [SR-2-03](spec-registry.md#sr-2-03) |
| カラーコード直書き | 一貫性崩壊 | [SR-2-02](spec-registry.md#sr-2-02) / [SR-2-08](spec-registry.md#sr-2-08) |

### 5.2 注意が必要

| パターン | 問題 | 対策 |
|---------|------|------|
| 3色以上のビビッドアクセント | 焦点散漫 | [SR-2-06](spec-registry.md#sr-2-06) |
| 類似色の組み合わせ | 差が不明瞭 | 色相差60度以上 |
| 背景色に近いアクセント | コントラスト不足 | [SR-9-01](spec-registry.md#sr-9-01) |
| グラデーション多用 | 安っぽい | タイトルスライドのみ |

## 6. グラデーションの使い分け

### 6.1 使ってよい場面

| 場面 | グラデーション | 適用方法 |
|------|-------------|---------|
| タイトルスライド背景 | --gradient-blue-pink | 全面背景 |
| CTA要素 | --gradient-blue-aqua | ボタン背景 |
| セクション区切り | --gradient-violet-pink | 帯状装飾 |
| カード背景（微細） | --gradient-subtle | overlay 5%透明度 |

### 6.2 使ってはいけない場面

| 場面 | 理由 | 代替 |
|------|------|------|
| 本文テキスト | 読みにくい | ソリッドカラー |
| 全スライド共通背景 | 特別感消滅 | --bg-dark |
| ボーダー | 安っぽい | ソリッド1色 |
| 3色以上のグラデーション | 視覚汚染 | 2色まで |

## 7. ダークテーマ対応

### 7.1 ダークテーマ時のカラー調整

| ライト | ダーク | 変更理由 |
|--------|--------|---------|
| --bg-dark: #fafafa | --bg-dark: #1F1F28 | 背景反転 |
| --fg: #43436c | --fg: #DCD7BA | テキスト反転 |
| vivid: そのまま | vivid: 彩度5%UP推奨 | 暗い背景で沈むため |

### 7.2 ダークテーマでのシャドウ

シャドウ変数の規則 → [SR-2-09](spec-registry.md#sr-2-09)

ダークテーマでは `box-shadow` の効果が弱まるため、**glow**（色付きシャドウ）に切替える。

```css
[data-theme="dark"] .card {
  box-shadow: var(--glow-blue);
}
```
