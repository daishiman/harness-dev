## 1. CARP原則（4つのデザイン基本原則）

Robin Williams『ノンデザイナーズ・デザインブック』に基づくスライド設計の基盤。

### 1.1 近接（Proximity）

**関連する要素を近づけ、無関連な要素を離す。**

| 状況 | NG | OK |
|------|----|----|
| アイコン+ラベル | gap: 24px（離れすぎ） | gap: 8px（--space-2） |
| カード見出し+本文 | margin: 32px | margin: 8px（--space-2） |
| セクション間 | margin: 16px（近すぎ） | margin: 48px（--space-7） |

**黄金ルール**: グループ内の間隔 < グループ間の間隔。**比率2倍以上**の差をつける。

```css
/* グループ内: 狭い */
.card-content { gap: var(--space-2); }  /* 8px */

/* グループ間: 広い */
.card-grid { gap: var(--space-5); }     /* 24px */
```

### 1.2 整列（Alignment）

**全要素に見えない線（軸）を意識する。**

| 整列タイプ | 適用場面 | 注意点 |
|-----------|---------|--------|
| 左揃え | 本文テキスト、リスト | デフォルトかつ最安全 |
| 中央揃え | タイトル、ヒーロー数値 | 3行以上のテキストには不向き |
| 右揃え | 数値、日付 | テキストには使わない |
| 両端揃え | 使わない | 日本語で不自然な空白が出る |

**原則**: 1スライド内の整列軸は**最大2本**。左揃えの軸 + 中央揃えの軸は共存可能。3本以上は散漫に見える。

```css
/* 左揃え軸 + 中央タイトル = 2軸でOK */
.slide-title { text-align: center; }
.slide-body { text-align: left; }
```

### 1.3 反復（Repetition）

**同種の要素は同じスタイルを繰り返す。**

| 要素 | 反復すべきスタイル |
|------|-----------------|
| カード群 | 同じサイズ・パディング・border-radius・シャドウ |
| リスト項目 | 同じフォントサイズ・アイコン色・行間 |
| セクション見出し | 同じウェイト・色・マージン |
| アクセント装飾 | 同じグラデーション・角丸 |

**原則**: 視覚的一貫性は**信頼感**を生む。1つのスライド内でカードのborder-radiusが12pxと16pxが混在するだけで「雑」に見える。

### 1.4 対比（Contrast）

**差をつけるなら大胆に。中途半端な差は「ミス」に見える。**

| 対比要素 | 弱すぎ（NG） | 適切 |
|---------|-------------|------|
| フォントサイズ | 1.6rem vs 1.8rem（差10%） | 1.6rem vs 3.0rem（差87%） |
| フォントウェイト | 400 vs 500（違い不明） | 400 vs 700（明確な差） |
| 色 | #4d699b vs #3B7DD8（微妙） | #888888 vs #3B7DD8（明確） |
| 余白 | 16px vs 20px | 16px vs 48px |

**黄金ルール**: 差をつけるなら**2倍以上の差**。差が2倍未満ならいっそ同じにする。

## 2. グリッドシステム

### 2.1 基本グリッド

16:9スライド（960×540論理単位）に対する推奨グリッド。

| グリッド | カラム数 | 用途 |
|---------|---------|------|
| 2カラム | 2等分 | 比較、Before/After |
| 3カラム | 3等分 | 3つの特徴、3ステップ |
| 4カラム | 4等分 | 4つのカード、マトリックス |
| 1+2 | 左1/3 + 右2/3 | 画像+テキスト |
| 2+1 | 左2/3 + 右1/3 | メインコンテンツ+サイドバー |

### 2.2 カラム幅の計算

```css
/* 利用可能幅の94%をコンテンツに使用（.slider__content max-width: 94%） */
/* gapを考慮した各カード幅 */

/* 2カラム: (94% - gap) / 2 */
.two-col { grid-template-columns: 1fr 1fr; gap: var(--space-5); }

/* 3カラム */
.three-col { grid-template-columns: repeat(3, 1fr); gap: var(--space-4); }

/* 4カラム */
.four-col { grid-template-columns: repeat(4, 1fr); gap: var(--space-3); }

/* 非対称: 1:2 */
.one-two { grid-template-columns: 1fr 2fr; gap: var(--space-5); }
```

### 2.3 カラム数の選択基準

| コンテンツ量 | 推奨カラム | 理由 |
|------------|-----------|------|
| 情報量少（1-2項目） | 1カラム中央 | 余白を活かしてインパクト |
| 比較（2項目） | 2カラム | 左右対比 |
| 概念分解（3-4項目） | 3-4カラム | 並列表示 |
| 詳細リスト（5項目以上） | 2行×3列 or リスト形式 | 4カラム以上は密度が高すぎる |

## 3. 余白のリズム

### 3.1 余白の3つの役割

| 役割 | 例 | 効果 |
|------|-----|------|
| 分離 | セクション間の大余白 | グループ境界を明示 |
| 呼吸 | テキスト周囲の余白 | 読みやすさ向上 |
| 強調 | ヒーロー要素周囲の大余白 | 孤立させて注目 |

### 3.2 スペーシングスケールの適用箇所

| 場所 | 推奨スペース | 変数 |
|------|------------|------|
| テキスト行間 | 1.5-1.8em | line-height |
| アイコン-ラベル間 | --space-2 | 8px |
| カード内padding | --space-4 ~ --space-5 | 16-24px |
| カード間gap | --space-5 ~ --space-6 | 24-32px |
| セクション間 | --space-7 ~ --space-8 | 48-64px |
| スライドpadding | 3%（.slider__item） | 固定 |
| タイトル-コンテンツ間 | --space-6 ~ --space-7 | 32-48px |

### 3.3 余白のアンチパターン

| NG | 理由 | 改善 |
|----|------|------|
| 全余白均一 | リズムがなく退屈 | 近接原則でメリハリ |
| 余白ゼロ | 窮屈、読めない | 最低--space-3は確保 |
| 余白過多 | 情報密度が低すぎる | コンテンツ量に合わせて調整 |
| 余白不規則 | 整列軸が崩れる | グリッドに揃える |

## 4. 視線誘導パターン

### 4.1 Z型パターン

```
①左上 ──→ ②右上
   ↙
③左下 ──→ ④右下
```

**適用**: 画像+テキスト混在スライド、最も汎用的

**実装のコツ**:
- ①にタイトル or ロゴ
- ②に視覚要素（画像・アイコン）
- ③にメインメッセージ
- ④にCTA or 結論

### 4.2 F型パターン

```
①左上 ──────→
②左中 ────→
③左下 ──→
```

**適用**: テキスト中心のスライド（リスト、説明）

**実装のコツ**:
- 各行の先頭にアイコンやビビッドカラーを配置
- 行の長さを段階的に短くすると自然なF型が生まれる

### 4.3 中央集中型

```
      ┌─────┐
      │  ●  │  ← フォーカルポイント
      └─────┘
   ↙    ↓    ↘
  補足  補足  補足
```

**適用**: ヒーロー数値、引用、キーメッセージ

**実装のコツ**:
- 中央要素は他の2倍以上のサイズ
- 周囲の要素はミュートカラー
- 周囲に--space-8以上の余白

### 4.4 逆N型（日本語最適化）

```
①右上
  ↓
②右下
  ↙
③左上
  ↓
④左下
```

**適用**: 日本語縦書き風のフロー、和風テーマ時

## 5. 黄金比・三分割法

### 5.1 三分割法（Rule of Thirds）

スライドを縦横3等分し、**交差点に重要要素を配置**する。

```
┌────┬────┬────┐
│    │    │    │
├────●────●────┤  ← 交差点4つ
│    │    │    │
├────●────●────┤
│    │    │    │
└────┴────┴────┘
```

**適用ルール**:
- タイトルは上1/3のライン上
- メイン画像は左2/3または右2/3
- 数値ハイライトは交差点に配置

### 5.2 60-30-10ルール（面積配分）

詳細は [color-strategy.md](color-strategy.md) §1、規則は [SR-2-06](spec-registry.md#sr-2-06) を参照。

**設計の意図**: アクセント（ビビッドカラー）の面積が増えると「強調」ではなく「塗りつぶし」になる。

## 6. スライドタイプ別レイアウト設計

### 6.1 情報量別の推奨レイアウト

| 情報量 | レイアウト | 理由 |
|--------|-----------|------|
| 極少（1メッセージ） | 中央1カラム、余白80% | インパクト最大化 |
| 少（2-3要素） | 2カラム or 中央縦積み | 余白を活かす |
| 中（4-6要素） | 2×2 or 3カラムグリッド | 整理された印象 |
| 多（7-9要素） | 3×3 or リスト | 密度に注意 |
| 極多（10以上） | 分割検討 or テーブル | 1スライドの限界 |

### 6.2 コンテンツ種別の配置

| コンテンツ | 推奨配置 | 理由 |
|-----------|---------|------|
| テキスト+画像 | 画像右、テキスト左 | F型に沿う |
| 数値+説明 | 数値上、説明下 | 視線の自然な流れ |
| Before/After | 左Before、右After | 左→右の進行方向 |
| フロー | 左→右 or 上→下 | 時間の進行方向 |
| 結論 | 中央 or 最下部 | 結語位置 |

## 7. 密度管理

### 7.1 密度の目安

| 指標 | 低密度 | 適正 | 高密度（危険） |
|------|--------|------|---------------|
| テキスト文字数 | 1-30字 | 30-80字 | 80字以上 |
| 要素数 | 1-3 | 3-6 | 7以上 |
| 余白率 | 70%以上 | 40-60% | 30%以下 |
| カラー数 | 1-2 | 2-3 | 4以上 |

### 7.2 密度が高すぎる場合の対処

1. **分割**: 1スライドを2スライドに分ける
2. **階層化**: 詳細をホバーツールチップに逃がす
3. **優先度カット**: L3情報を削除
4. **テーブル化**: 並列情報はテーブルで整理

## 8. 構図チェックリスト

### 必須チェック

- [ ] 近接：グループ内間隔 < グループ間間隔（2倍以上の差）
- [ ] 整列軸が2本以内
- [ ] 同種要素のスタイル反復
- [ ] 対比は2倍以上の差
- [ ] グリッドカラム数がコンテンツ量に適切

### 品質チェック

- [ ] 三分割法の交差点にフォーカルポイントがあるか
- [ ] アクセント面積 → [SR-2-06](spec-registry.md#sr-2-06)
- [ ] 余白率が40-60%か
- [ ] テキスト80字 / 要素6個を超えていないか
- [ ] カード間gap > カード内padding

## 9. ナラティブ構造パターン（SR-06, 2026-05-09 追加）

`schema: meta.narrative_pattern` の選択肢に対応するデッキ全体の物語構造。各スライドではなく**デッキ全体の章配分**を決める軸。

### 9.1 `purpose-background-means-conclusion`（推奨デフォルト）

**目的 → 背景 → 手段 → 結論** の 4 ブロック構成。

```
[目的: 1-2枚]    なぜこの話をするか / ゴール定義
   ↓
[背景: 3-6枚]    現状・課題・前提認識の共有
   ↓
[手段: 6-12枚]   解決策の提示・要素分解・具体例
   ↓
[結論: 2-4枚]    要約・行動喚起・Q&A 誘導
```

- **適合**: 業務報告・コンサル提案・社内研修
- **章数**: 4（または 3〜5、結論章は短く）
- **検証**: 各章のスライド配分が `背景 ≧ 手段 ≧ 結論 ≧ 目的` または `手段 ≧ 背景 ≧ 結論 ≧ 目的`。手段が肥大しすぎる場合は再分割（参考: 04-09 E-02）

### 9.2 `z-shape`（Z 字構造）

**導入 → 展開 → 転換 → 結論** の 4 ブロック。視線の Z 字移動になぞらえた古典構造。

```
[導入]→[展開]
        ↘
   [転換]→[結論]
```

- **適合**: 短時間（15分以下）プレゼン・キーノート的演出
- **章数**: 4 固定。各章 3-5 枚に収める

### 9.3 `prep`（PREP 法）

Point → Reason → Example → Point。1 スライド〜短時間プレゼンの定番。

- 詳細: `references/diagram-business.md` PREP 型
- スライド枚数: 4-8 枚程度。デッキ全体に適用する場合はセクションごとに mini-PREP 化

### 9.4 `fabe`（FABE フレーム）

Feature → Advantage → Benefit → Evidence。商品・サービス紹介向け。

- 詳細: `references/diagram-fabe.md`
- 章数: 4 + 表紙・結論で計 6 章程度

### 9.5 `scqa`（SCQA / Pyramid Principle）

Situation → Complication → Question → Answer。コンサル提案の王道。

- **適合**: 経営層向け 30-60 分プレゼン
- 章数: 4 + 詳細展開で計 5-7 章

### 9.6 `before-after`

現状（Before） → 変化要因 → 改善後（After）。

- **適合**: 改善提案・導入事例
- 章数: 3〜4

### 9.7 `problem-solution`

問題定義 → 原因分析 → 解決策 → 効果。

- **適合**: 課題解決提案
- 章数: 4

### 9.8 `custom`

上記いずれにも当てはまらない独自構成。`structure.md` で明示的に章構成を文書化することを必須とする。

## 10. ペルソナ別導線パターン（SR-11, 2026-05-09 追加）

聴衆ペルソナ（経営層 / 現場担当 / エンジニア / 非エンジニア）によって、同じテーマでも順序・粒度が変わる。デッキ前半に**ペルソナ判別スライド**を 1 枚入れ、聴衆が「自分向けのパート」を予測できるようにする。

### 10.1 経営層向け導線（短時間・結論先出し）

```
[結論サマリ] → [3 つの根拠] → [投資対効果] → [次の一手]
```

- 1 章あたり 2-3 枚に圧縮
- 数値・グラフ中心、技術詳細は appendix へ
- スライド総数 8-15 枚
- 推奨 narrative_pattern: `scqa` または `purpose-background-means-conclusion` の「目的・結論」短縮版

### 10.2 現場担当向け導線（具体例・手順重視）

```
[共感: 現場の困りごと] → [全体像] → [具体手順] → [失敗パターン] → [明日からの一歩]
```

- 1 章あたり 4-6 枚
- スクリーンショット・操作画面を多用
- スライド総数 25-40 枚
- 推奨 narrative_pattern: `before-after` または `problem-solution`
- **失敗パターン提示型**スライド（→ `slide-design-patterns.md`）を 1-2 枚必ず入れる

### 10.3 エンジニア向け導線（仕組み・設計判断）

```
[アーキテクチャ図] → [設計判断と理由] → [実装サンプル] → [トレードオフ] → [拡張ポイント]
```

- コードブロック・図解中心、抽象論は最小限
- 推奨 narrative_pattern: `purpose-background-means-conclusion`

### 10.4 非エンジニア向け導線（用語先出し・比喩）

```
[用語の予告] → [日常の比喩] → [構成要素] → [手順] → [明日試せる行動]
```

- **用語注釈先出しスライド**（→ `slide-text-guidelines.md`）を冒頭で 1 枚使う
- カード見出しは `1.85rem` 以上、本文 `1.45rem` 以上
- 推奨 narrative_pattern: `before-after` または `purpose-background-means-conclusion`

### 10.5 混合ペルソナ向け（経営+現場 併存）

経営層と現場担当が同席する場合、**ペルソナ別パートを章で分ける**。

```
[共通: 全体像 1 章] → [経営パート 1 章] → [現場パート 1 章] → [共通: 行動 1 章]
```

- 各パートの開始時にペルソナ宣言スライドを入れる（「ここからは現場担当向け」等）
- 各章 3-5 枚で統一して肥大を防ぐ（参考: 04-09 E-02 章肥大エスカレーション）
