## 1. SVG設計メモの必須項目（12項目）

12項目記載必須は [SR-5-07](spec-registry.md#sr-5-07) で定義。各項目の意味・例:

| # | 項目 | 説明 | 例 |
|---|------|------|-----|
| 1 | **viewBox** | SVG全体の座標空間 | `"0 0 860 480"` |
| 2 | **要素配置座標** | 各カード・ノードのx,y座標 | カード1: x=22、カード2: x=228 |
| 3 | **カードサイズ** | 幅×高さ + 角丸rx | w=190px、h=220px、rx=14 |
| 4 | **フォントサイズ** | CSS変数名 + rem値 | fs-small(1.4rem)、fs-body(1.6rem) |
| 5 | **最大文字数/行** | 1行あたりの文字数上限 | **1行最大10文字** |
| 6 | **最大行数** | テキスト要素の最大行数 | 最大2行 |
| 7 | **テキスト改行位置** | 全テキストの明示的な改行位置と文字数 | 「業務の流れや、」(7文字) / 「時間がかかっている」(9文字) |
| 8 | **padding** | カード内余白 | padding: 16px |
| 9 | **gap** | カード間・要素間の余白 | gap=20px |
| 10 | **接続線/矢印** | 線の始点・終点座標、stroke仕様 | line(350,250)→(350,160)、stroke-width: 3 |
| 11 | **アクセントカラー** | 各要素に適用する色 | accent-blue、accent-aqua |
| 12 | **テキスト文字数検証** | 全テキスト要素の文字数カウント結果 | ①「議事録の下書き」(7文字) → OK |

## 2. テキスト収まり計算ロジック

### 2.1 カード有効幅の算出

```
カード有効幅 = カード幅 - (左padding + 右padding) - アイコン領域幅
```

例: カード幅260px、padding 12px×2、アイコン円r=14+余白8px = 36px の場合
→ 有効幅 = 260 - 24 - 36 = **200px**

### 2.2 1行あたりの最大文字数

```
最大文字数 = floor(カード有効幅 / フォントサイズpx値)
```

フォントサイズの参照表:

| CSS変数 | rem値 | px近似値（1rem≒16pxベース想定） | 日本語1文字幅の目安 |
|---------|-------|------|------|
| fs-title | 3.6rem | ≒58px | ≒58px |
| fs-heading | 2.8rem | ≒45px | ≒45px |
| fs-subheading | 2.0rem | ≒32px | ≒32px |
| fs-body | 1.6rem | ≒26px | ≒26px |
| fs-small | 1.4rem | ≒22px | ≒22px |

**ただしSVG内テキストは `<text>` 要素のため、px値で制御する**（[SR-1-05](spec-registry.md#sr-1-05)）。

SVG内のfont-sizeは以下のpx値を標準とする:

| 用途 | SVG font-size | 日本語1文字幅 |
|------|--------------|-------------|
| タイトル | 28px | ≒28px |
| サブタイトル | 22px | ≒22px |
| 本文 | 18px | ≒18px |
| 補足・ラベル | 14px | ≒14px |
| 注記 | 13px | ≒13px |
| 小バッジ・角ラベル | 12px | ≒12px |

最小フォントサイズ制約 → [SR-3-05](spec-registry.md#sr-3-05)（13px以上原則、12px は小バッジ・軸ラベルのみ許容）。

**計算例**:
- カード有効幅 200px、font-size 14px → 200 / 14 ≒ **14文字/行**
- ただし安全マージンとして **実運用は計算値の70〜80%** を上限とする
- 上記例: 14 × 0.75 ≒ **10文字/行** を最大文字数とする

### 2.3 必要行数の検証

```
必要行数 = ceil(テキスト文字数 / 1行最大文字数)
```

- 必要行数 ≤ 最大行数 → OK
- 必要行数 > 最大行数 → **テキストのリライト** or **カードサイズの拡大** が必要

## 3. テキスト改行位置の指定方法

### 3.1 記述フォーマット

各テキスト要素について、以下の形式で改行位置と文字数を明示する:

```
- ①「業務の流れや、」(7文字) / 「時間がかかっている」(9文字) / 「仕事を教えていただく」(10文字)
```

### 3.2 改行位置の判断基準

| 優先度 | 改行位置 |
|--------|---------|
| 1 | 読点「、」の直後 |
| 2 | 助詞（を、が、に、で、は、と）の直後 |
| 3 | 意味の切れ目（主語と述語の間、修飾語と被修飾語の間） |
| 4 | 最大文字数に達する位置（強制改行、非推奨） |

### 3.3 SVG実装時の改行

`<tspan>` 改行と `dy` 算出規則 → [SR-5-04](spec-registry.md#sr-5-04)（dy = font-size × 1.5）。

```svg
<text x="100" y="60" font-size="14">
  <tspan x="100" dy="0">業務の流れや、</tspan>
  <tspan x="100" dy="20">時間がかかっている</tspan>
  <tspan x="100" dy="20">仕事を教えていただく</tspan>
</text>
```

- font-size 14px → dy ≒ 20px
- font-size 18px → dy ≒ 28px

## 4. viewBox設計パターン

### 4.1 標準viewBox算出式

算出式の正本 → [SR-5-02](spec-registry.md#sr-5-02)。高さは「ヘッダー高さ + カード高さ + 下部テキスト高さ + 上下マージン」。viewBox 解像度標準 → [SR-1-02](spec-registry.md#sr-1-02) / [SR-5-01](spec-registry.md#sr-5-01)。

### 4.2 よく使うviewBoxパターン

| レイアウト | viewBox | 計算根拠 |
|-----------|---------|---------|
| 1×3カード(260px) | "0 0 860 440" | 260×3 + 24×2 + 28×2 = 860 |
| 1×4カード(190px) | "0 0 840 300" | 190×4 + 16×3 + 22×2 = 852→840 |
| 1×5カード(170px) | "0 0 960 420" | 170×5 + 14×4 + 27×2 = 960 |
| 2カラム対比(380px) | "0 0 860 480" | 380×2 + 40 + 20×2 = 860 |
| 円形サイクル | "0 0 700 700" | 中心(350,350)、半径220+ノード幅80 |
| 同心円 | "0 0 700 500" | 中心(350,230)、最大半径220+余白 |
| ロジックツリー | "0 0 900 600" | ルート幅260+子ノード260×3+gap |
| 縦型プロセス | "0 0 600 600" | 幅=進捗バー8+gap+カード460+余白 |

## 5. スライドタイプ別チェックリスト

### 5.1 横並びカード型（1×N）

- [ ] viewBox幅 = カード幅×N + gap×(N-1) + 左右マージン
- [ ] 全カードのx座標を明記
- [ ] カード内テキストの文字数 ≤ (有効幅/font-size) × 0.75
- [ ] 全テキストの改行位置を文字数付きで明示
- [ ] hover効果（translateY、shadow変化）を記載

### 5.2 対比型（2カラム）

- [ ] viewBox幅 = パネル幅×2 + gap + 左右マージン
- [ ] 左右パネルのx座標、accent-colorを明記
- [ ] featured側の強調仕様（border、scale、リボンバッジ）
- [ ] 各項目のテキスト文字数検証（パネル有効幅から算出）
- [ ] 左右の項目数が異なる場合のpadding調整

### 5.3 サイクル型（円形配置）

- [ ] viewBox = 正方形（中心座標、半径、ノード幅から算出）
- [ ] 各ノードの角度と座標（cos/sinで計算）
- [ ] ノード矩形のサイズとtranslateオフセット
- [ ] 弧矢印のパスコマンド形式（A arc指定）
- [ ] マーカー定義（`<defs>` 内の矢じり）
- [ ] 中央ラベルの仕様

### 5.4 タイムライン型（縦型）

- [ ] 中央線のx座標、y範囲
- [ ] 各ステージ円のcy座標
- [ ] カードの左右交互配置ルール（x座標）
- [ ] カード内テキストの文字数（カード幅から算出）

### 5.5 フロー型（分岐）

- [ ] 上部ノードのサイズと中央配置
- [ ] 分岐線のパスと分岐点の座標
- [ ] 下部ノード群の均等配置x座標
- [ ] 各ノード内テキストの文字数制限

### 5.6 シェブロン型

- [ ] 各シェブロンのx座標とpolygon形状
- [ ] **有効幅 = シェブロン幅 - 矢印尖端幅 - 左右padding** から文字数算出
- [ ] 各ステップの説明テキスト改行位置

### 5.7 三角サイクル型

- [ ] 三角形の中心座標と半径
- [ ] 3ノードの座標（正三角形: 上、左下、右下）
- [ ] ノード間弧矢印

### 5.8 バリュープロポジション型

- [ ] 中央ダイヤモンドのサイズと座標
- [ ] 3方向カードの配置座標
- [ ] 中央→カード接続線の座標

## 6. 品質検証基準

structure.md出力後、以下を検証する:

| # | 検証項目 | 基準 | 不合格時の対応 |
|---|---------|------|-------------|
| Q1 | viewBox有無 | 全SVGスライドにviewBox記載あり | viewBoxを追加 |
| Q2 | 座標の整合性 | 要素がviewBox内に収まる | 座標またはviewBoxを修正 |
| Q3 | テキスト文字数 | 全テキストが最大文字数以内 | テキストをリライト |
| Q4 | テキスト行数 | 全テキストが最大行数以内 | 改行位置を再設計 |
| Q5 | 改行位置の明示 | 全テキスト要素に改行位置と文字数が記載 | 改行位置を追記 |
| Q6 | padding/gap | 全カード型にpadding、全並列型にgap記載 | 数値を追記 |
| Q7 | フォントサイズ | 全テキストにCSS変数名+値が記載 | フォント仕様を追記 |
| Q8 | accent-color | 全要素にアクセントカラー指定あり | カラーを追記 |
