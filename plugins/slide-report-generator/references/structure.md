# スライド構成テンプレート

## 概要

プレゼンテーションは「理想→現実→ギャップ→解決策」のサイクルを基本構造とする。
このサイクルは**全体構成**と**各セクション**の両方で適用する。

---

## 基本サイクル（Gap-Bridge Cycle）

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│    ① 理想（To-Be）                                              │
│         │                                                       │
│         ▼                                                       │
│    ② 現実（As-Is）                                              │
│         │                                                       │
│         ▼                                                       │
│    ③ ギャップの可視化                                            │
│         │                                                       │
│         ├──→ ③-a: このままだとどうなるか（リスク・問題）          │
│         │                                                       │
│         └──→ ③-b: 理想に近づく途中の障壁                         │
│         │                                                       │
│         ▼                                                       │
│    ④ 解決策（方法・手順・ツール）                                 │
│         │                                                       │
│         ▼                                                       │
│    ⑤ まとめ（成果・次のサイクルへ）                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 全体構成（マクロサイクル）

全体のプレゼンテーションを1つの大きなサイクルとして構成する。

| Phase | スライドタイプ | 内容 | 推奨枚数 |
|-------|---------------|------|---------|
| **導入** | title, agenda | タイトル、自己紹介、目次 | 2-3枚 |
| **① 理想** | section, message, stats | あるべき姿、目標、理想の状態 | 3-5枚 |
| **② 現実** | section, list, compare | 現状の課題、データ、実態 | 3-5枚 |
| **③ ギャップ** | compare, stats, diagram | 理想と現実の差、問題点 | 2-4枚 |
| **③-a リスク** | message, list | このまま放置した場合の結末 | 1-2枚 |
| **③-b 障壁** | list, diagram | 理想に向かう際の困難 | 1-2枚 |
| **④ 解決策** | section, flow, list, diagram | 方法、手順、ツール、プロセス | 5-8枚 |
| **⑤ まとめ** | section, message, stats | 成果、期待効果、アクション | 2-3枚 |
| **CTA** | message, title | 行動喚起、連絡先 | 1-2枚 |

### 推奨スライド数: 20-35枚

---

## セクション構成（ミクロサイクル）

各セクション内でも同じサイクルを小さく回す。

### セクション内構成パターン

```
【セクション: 例「業務効率化」】

1. セクションヘッダー（slide-section）
   └─ テーマの導入

2. 理想の提示（slide-message / slide-stats）
   └─ 「業務時間を50%削減できたら」

3. 現実の提示（slide-list / slide-compare）
   └─ 「現状は手作業で8時間/日」

4. ギャップの可視化（slide-compare / slide-diagram）
   └─ 「4時間の差 = 年間1000時間の損失」

5. 解決策の提示（slide-flow / slide-list）
   └─ 「自動化ツールの導入3ステップ」

6. セクションまとめ（slide-message）
   └─ 「これで業務時間50%削減を実現」
```

## スライドタイプ別用途

### ① 理想（To-Be）向けスライド

| スライドタイプ | 用途 | 効果 |
|---------------|------|------|
| `slide-message` | 理想状態の一言表現 | 印象付け、感情喚起 |
| `slide-stats` | 目標数値の提示 | 具体性、説得力 |
| `slide-quote` | 権威者の言葉 | 信頼性 |

### ② 現実（As-Is）向けスライド

| スライドタイプ | 用途 | 効果 |
|---------------|------|------|
| `slide-list` | 現状の課題列挙 | 網羅性 |
| `slide-stats` | 現状のデータ | 客観性 |
| `slide-table` | 詳細な現状分析 | 精密さ |

### ③ ギャップ向けスライド

| スライドタイプ | 用途 | 効果 |
|---------------|------|------|
| `slide-compare` | Before/After対比 | 差の可視化 |
| `slide-diagram` | 構造的な差の図解 | 理解促進 |
| `slide-chart` | 数値差のグラフ化 | 視覚的インパクト |

### ④ 解決策向けスライド

| スライドタイプ | 用途 | 効果 |
|---------------|------|------|
| `slide-flow` | 手順・プロセス | 実行可能性 |
| `slide-list` | 必要なツール・方法 | 具体性 |
| `slide-diagram` | システム構成図 | 全体像の把握 |
| `slide-timeline` | スケジュール | 計画性 |

### ⑤ まとめ向けスライド

| スライドタイプ | 用途 | 効果 |
|---------------|------|------|
| `slide-message` | 結論の一言 | 記憶定着 |
| `slide-stats` | 期待成果の数値 | 説得力 |
| `slide-list` | 次のアクション | 行動喚起 |

```
【導入】3枚

 2. 自己紹介/会社紹介（slide-message）
 3. アジェンダ（slide-agenda）

【① 理想】4枚
 4. セクション: 理想の未来（slide-section）
 5. あるべき姿（slide-message）
 6. 目標数値（slide-stats）
 7. 成功事例（slide-quote）

【② 現実】4枚
 8. セクション: 現状の課題（slide-section）
 9. 課題一覧（slide-list）
10. 現状データ（slide-stats）
11. 詳細分析（slide-table）

【③ ギャップ】4枚

14. このまま放置すると（slide-message）※リスク

【④ 解決策】10枚
16. セクション: 解決への道筋（slide-section）
17. 全体フロー（slide-flow）
18. ステップ1詳細（slide-list）
19. ステップ2詳細（slide-list）
20. ステップ3詳細（slide-list）
21. 必要なツール（slide-diagram）
22. 導入スケジュール（slide-timeline）
23. 期待効果（slide-stats）
24. 事例紹介（slide-image）
25. Q&A想定（slide-table）

【⑤ まとめ】3枚
26. セクション: まとめ（slide-section）
```

```
【全体（マクロサイクル）】
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  導入 → 理想 → 現実 → ギャップ → 解決策 → まとめ → CTA     │
│            │                        │                      │
│            ▼                        ▼                      │
│     ┌──────────────┐        ┌──────────────┐              │

│     │  →ギャップ   │        │  →ギャップ   │              │
│     │  →解決策    │        │  →解決策    │              │

│                                                            │

```

## ベストプラクティス

### すべきこと

- 各サイクルの「理想」と「現実」に具体的な数値を入れる
- ギャップを感情的にも論理的にも訴求する
- 解決策は実行可能な粒度まで分解する
- セクション開始時にミクロサイクルの予告をする
- まとめでマクロサイクルの全体像を振り返る

### 避けるべきこと

- いきなり解決策から始める（理想・現実・ギャップなし）
- ギャップを曖昧にする（「なんとなく問題」）
- 解決策が抽象的すぎる（「頑張りましょう」）
- サイクルが途中で切れる（現実→解決策で終わる）
