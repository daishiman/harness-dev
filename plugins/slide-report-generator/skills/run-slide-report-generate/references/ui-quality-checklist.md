# UI品質チェックリスト（ui-quality-reviewer 検証基準 SSOT）

> **正本**: このファイルは ui-quality-reviewer から抽出した手続き知識/規範の SSOT。run-slide-report-generate の SKILL.md と agent 本体（agents/ui-quality-reviewer.md）の双方がこれを参照する。規則の上位正本 (SR-ID) は spec-registry.md を辿る。

**責務**: UI品質検証のドメイン定義（用語集・評価基準・制約カタログ CONST_001-007）と検証基準（多面検証 MUST/SHOULD/MAY チェックリスト・必須構造検証 S1〜S26・多面検証チェックリスト・テーマ別視覚検証ポイント・修正指針・よくある問題と対処法）の逐語正本。ui-quality-reviewer（薄化アダプタ）は役割・起動条件・I/O契約に専念し、詳細規範は本 reference を SSOT とする。

## ビジネスルール（制約カタログ CONST_001-007）

- **CONST_001 (必須構造検証ゲート)**: S1〜S26 に1つでも違反があれば UI 品質レビューを中断し html-generator へ差し戻す。
  - 目的: 構造的欠陥を含むスライドが下流（deck-evaluator・配布）へ流出するのを防ぐ。
  - 背景: CSS/JS分離違反・コンテンツ欠落・印刷崩れは局所修正では救済できず、生成段階の差し戻しが最も低コストである。
- **CONST_002 (チェックリスト全件消化)**: S1〜S26 と検証チェックリストの全項目を、記憶や主観に頼らず機械的に1件ずつ判定する。
  - 目的: 検証漏れ（人が「たぶん大丈夫」で飛ばす項目）をゼロにする。
  - 背景: The Checklist Manifesto の知見（チェックリストの価値は省略を構造的に許さない点にある）。
- **CONST_003 (絵文字禁止・Font Awesome統一)**: スライド本文・SVG `<text>` 内を含め絵文字を使用せず、アイコンは `fa-*`（Font Awesome）で表現する。SVG内 `&#xf` PUAコードは禁止（S23）。
  - 目的: 対外資料の一貫した品位と、CDN未ロード時のアイコン消失防止を両立する。
  - 背景: 絵文字はOS/フォント依存でレンダリングが揺れ、SVG内FA unicodeはCDN依存で全消失リスクがある（実運用で発生）。
- **CONST_004 (最小フォントサイズ)**: 本文相当テキストは 1.6rem 以上、絶対下限は 1.4rem。SVG `<text>` は 13px 以上（補助ラベルのみ12px許容、11px以下禁止）。
  - 目的: 対面大画面・プロジェクター投影での可読性を保証する。
  - 背景: 11px以下のSVGテキストは大画面で不可視という実測知見（S22）。
- **CONST_005 (CSS/JS外部分離)**: index.html にインライン `<style>` / `<script>` を置かず、styles.css / scripts.js を外部参照する（S1/S2）。
  - 目的: 検証・修正・再利用の単位を3ファイルに固定し、保守性と差分検証性を保つ。
  - 背景: インライン化は GSAP clearProps によるスタイル破壊や差分検証困難を招いた（再発防止）。
- **CONST_006 (GSAP clearProps安全パターン)**: clearProps は content.children のみに適用し、`querySelectorAll('*')` への適用を禁止。foreignObject内divは `.fo-card`（/ `.fo-card--row`）で保護する（S17/S18）。
  - 目的: SVG fill/stroke破壊・foreignObjectレイアウト崩壊を防ぐ。
  - 背景: 全要素clearPropsとインラインstyle依存が描画崩壊を起こした実例（v6.3.0で対策）。
- **CONST_007 (画面=印刷一致のA4仕様)**: 印刷CSSは単位 mm/rem/vw を用い px を使わず、`@page { margin: 0 }`・A4フルサイズ（297mm×210mm）で画面と印刷を一致させる（S7/S8/S19）。
  - 目的: 印刷時のカード消失・余白崩れ・内容欠落を防ぐ。
  - 背景: box-shadow依存や固定px指定で印刷時にカードが消える事故が起きた。

## 多面検証チェックリスト（MUST / SHOULD / MAY）

出力の「品質基準（出力に必ず含む必須フィールド）」を満たすため、以下の多面検証チェックリストを全件消化する。

必須（MUST）チェックリスト 全テーマ共通:

- [ ] すべてのテキストが切れていない
- [ ] 16:9アスペクト比が維持されている
- [ ] フォントサイズが1.4rem以上
- [ ] WCAG AA基準（コントラスト比4.5:1以上）を満たす
- [ ] ナビゲーション・ボタンが視認できる
- [ ] クリック/タップ領域が44x44px以上
- [ ] スライド内統一感: カラーコード直書きなし（CSS変数使用）
- [ ] スライド内統一感: アクセントカラーは2色以内
- [ ] スライド内統一感: 同レベル要素のスタイルが揃っている

必須（MUST）ライトモード:

- [ ] CSS変数が正しく設定（--bg-dark: #FFFFFF, --fg: #2D2D2D）
- [ ] カード・ボックスに適切な影またはボーダー
- [ ] 印刷時も内容が読める

必須（MUST）ダークモード:

- [ ] CSS変数が正しく設定（--bg-dark: #1F1F28, --fg: #DCD7BA）
- [ ] 純黒・純白を使用していない
- [ ] プロジェクター投影時のコントラスト確保

推奨（SHOULD）:

- [ ] 各スライドの視覚的バランスが良い
- [ ] 適切な余白が確保されている
- [ ] ホバーエフェクトが明確（0.2s以内のトランジション）
- [ ] アニメーションがスムーズ（60fps維持）
- [ ] フォーカス状態が視覚的に明確

任意（MAY）:

- [ ] ツールチップの追加
- [ ] 追加のビジュアル装飾
- [ ] 印刷用スタイルの最適化
- [ ] prefers-color-schemeによる自動テーマ切替

## 検証基準（必須構造検証 S1〜S26 と多面検証）

各基準は第三者が合否判定できる客観条件で記述し、agent の完了チェックリスト（5.3）はこれらを全件消化することで充足する。必須構造検証は他の検証に優先し、1つでも違反があれば UI 品質レビューを中断し html-generator へ差し戻す（CONST_001）。

### 必須構造検証基準 S1〜S26（最優先）

HTML生成後、他のUI検証に先立ち以下を**必ず**確認する。1つでも違反があれば UI 品質レビューを中断し html-generator へ差し戻す。各基準は第三者が合否判定できる客観条件で記述する。

| # | 検証項目 | 基準（検証可能条件） | 検出方法 |
|---|---------|------|----------|
| S1 | CSS/JS分離 | index.htmlに`<style>`タグおよびインライン`<script>...</script>`が0件 | `<style>`・インライン`<script>`の有無を確認 |
| S2 | 外部ファイル参照 | `<link rel="stylesheet" href="styles.css">` と `<script src="scripts.js"></script>` が各1件以上存在 | link/scriptタグの確認 |
| S3 | 質問スライド配置順序 | 各セクション内で質問スライド(`.question-badge`)が背景情報スライドの後に配置 | スライド順序の確認 |
| S4 | 質問スライドフォント | styles.cssに `.question-badge ~ .main-message { font-size: var(--fs-subheading) }` が存在 | CSSルールの確認 |
| S5 | slide-area要素 | `.slider` > `.slide-area` > `.slider__container` の3層構造が存在 | HTML構造の確認 |

| S8 | 印刷A4フルサイズ | `@page { margin: 0 }` + `.slider__item { width: 297mm; height: 210mm; border: none }` が存在 | @media print内の確認 |
| S9 | ページネーション色分け | 5の倍数にmilestone-5、10の倍数にmilestone-10クラス付与 | scripts.js buildNavigation()の確認 |

| S15 | CARP原則 | 近接（グループ内gap<グループ間gap）かつ対比（差2倍以上）が適用 | gap/margin/font-size値の確認 |
| S16 | 60-30-10配色 | アクセントカラー面積が10%以下、1スライドのビビッドカラーが2色以内 | 色使用箇所の確認 |
| S17 | clearProps安全パターン | clearPropsがcontent.childrenのみに適用。`querySelectorAll('*')`でのclearProps適用が0件（SVG fill/stroke破壊・foreignObjectレイアウト崩壊の原因） | scripts.jsのclearProps使用箇所を確認 |
| S18 | foreignObject CSS保護 | foreignObject > divに`class="fo-card"`（またはfo-card fo-card--row）が適用。インラインstyleのみのレイアウト定義が0件（GSAPのclearPropsで破壊される） | index.htmlのforeignObject内div要素のclass属性を確認 |
| S19 | A4印刷仕様準拠 | structure.mdのA4印刷品質保証仕様（単位mm/rem/vw、px禁止、@page margin:0、画面=印刷一致）がstyles.cssに実装 | @media printセクションの単位・レイアウト確認 |

### 多面検証チェックリスト

各項目の基準は第三者が合否判定できる客観条件で記述する。

#### テキスト品質

| 検証項目 | 基準 | 問題検出方法 |
|---------|------|-------------|
| テキスト切れ | すべてのテキストが表示領域内に収まる（overflow切れ0件） | overflow: hidden で切れていないか |

| 要素 | 最小サイズ | 推奨サイズ | CSS変数 |
|------|-----------|-----------|---------|
| メインタイトル | 3.6rem | 4.8rem | --fs-title |
| セクション見出し | 2.4rem | 3.2rem | --fs-heading |

### テーマ別視覚検証ポイント

スクリーンショット目視で各テーマの視認性を確認する観点。

| 検証項目 | ライトモード | ダークモード |
|---------|------------|-------------|
| 背景との対比 | 白背景で薄くないか | 暗い背景で埋もれていないか |
| カードの浮き | 影・ボーダーで区別 | 背景より明るい色で区別 |
| ボタン視認性 | 青系で目立つか | 明るい色で目立つか |
| 印刷プレビュー | 白背景で正常か | - |
| プロジェクター確認 | - | コントラスト十分か |

### 修正指針（検出問題→修正方法）

検出問題ごとの修正方法。font-size 縮小は CONST_004 の最小値（1.4rem）以上の範囲でのみ行う。

| 問題 | 修正方法 |
|------|----------|
| テキスト切れ | max-width拡大、テキスト簡略化（font-size縮小は最小値以上のみ） |
| 不自然な改行 | `<br>`タグ挿入、max-width調整 |
| バランス問題 | padding/margin調整、flex設定修正 |
| 色問題 | CSS変数使用に統一 |
| スペーシング | gap/margin調整 |
| 文字が小さすぎる | CSS変数を使用（--fs-body等）、最小1.4rem以上に |
| コントラスト不足 | テーマに応じた色変数を使用 |

## よくある問題と対処法

### テキスト関連

| 問題 | 原因 | 対処法 |
|------|------|--------|
| カード内テキスト切れ | max-width不足 | `max-width: 280px` → `max-width: 350px` |
| 統計値の切れ | フォントサイズ大 | `--fs-title` → `--fs-heading` + `white-space: nowrap` |
| 長いタイトル切れ | 固定幅 | `<br>`で明示改行、またはmax-width拡大 |

### レイアウト関連

| 問題 | 原因 | 対処法 |
|------|------|--------|
| カードの高さ不揃い | align-itemsなし | `align-items: stretch`追加 |
| フローステップの詰まり | gap不足 | `gap: 1.5rem` → `gap: 2rem` |
| 比較パネルのずれ | flex設定 | `flex: 1`と`max-width`を両方設定 |

### ライトモード固有

| 問題 | 原因 | 対処法 |
|------|------|--------|
| テキストが薄い | --fg-dim使用 | メインテキストは`--fg`使用 |
| カードが見えにくい | 背景色が近い | `border`追加または`box-shadow`追加 |
| ボタンが目立たない | 半透明背景 | `border`を明確に、ホバー時の変化を強調 |
| 印刷時にカードが消える | box-shadow依存 | border追加で輪郭を明示 |
| アクセント色が弱い | コントラスト不足 | 彩度を上げる or 背景を調整 |

### ダークモード固有

| 問題 | 原因 | 対処法 |
|------|------|--------|
| テキストが眩しい | #FFFFFF使用 | --fg（#DCD7BA）に変更 |
| カードが浮いて見えない | 背景色が近い | 背景より1段階明るく（#363646） |
| ホバー状態がわからない | 色変化が小さい | 明度差を大きく or グロー追加 |
| プロジェクターで見えない | コントラスト不足 | 明度差を15%以上に |
| アイコンが暗い | 同系色 | 明るい色（#7E9CD8等）使用 |

### UX共通問題

| 問題 | 原因 | 対処法 |
|------|------|--------|
| クリック領域が小さい | padding不足 | 最小44x44px確保 |
| フォーカスが見えない | outline未設定 | outline: 2px solid var(--accent) |
| 状態変化がわからない | transition未設定 | transition: 0.2s ease追加 |
| スクロールがガタつく | scroll-snap未設定 | scroll-snap-type: x mandatory |
