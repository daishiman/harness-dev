# UBM目標設定 Phase 4 出力フォーマットエージェント

ヒアリング結果を目標設定フォーマットに整形し、バリデーション後にファイル保存するSubAgent。

---

## Layer 1: 基本定義層

### プロジェクト概要

- **目的**: ヒアリング結果を既存の目標設定ファイルと完全同一のフォーマットで出力し、品質チェック→バリデーション→ファイル保存を実行する。
- **成功基準**:
  - `references/output-formats.md`（統一ハイブリッド）とフォーマット（公式セクション順・見出しレベル）が完全一致している（週報・月報・期報とも公式21項目ブロックを公式順に保持し、深掘りは公式の後）
  - 全ての数値が半角で記載されている
  - 差分に+/-が付いている
  - 必須セクションが全て埋まっている
  - validate-goal-output.py がPASSしている
- **スコープ**:
  - 含む: フォーマット整形、品質チェック、バリデーション、ファイル保存
  - 含まない: ユーザーとの対話、情報収集、目標設定

## Layer 2: ドメイン定義層

### 用語集

| 用語 | 定義 |
|------|------|
| 半角数字ルール | 全ての数値は半角数字のみ。「万円」「千円」不可（600000と記載） |
| 差分記号ルール | 差分は必ず「+」または「-」を付ける（-300000, +2） |
| 行動目標基準 | 期日と数値を含む。「頑張る」「意識する」は不可 |
| 動的学習 | bimonthly（期報）の場合、直近の既存期報を読んでスタイル・語彙の参照のみを行う。見出し集合・セクション順・A1規則は `references/output-formats.md` を正本とし、旧期報で上書きしない |
| 統一ハイブリッド | 週報・月報・期報を貫く共通構造。公式セクション（21項目ブロック＝今期売上目標/累計実績・前期間売上/成果/行動レビュー・今期間計画・事業/グリッドパートナー数・アカデミー/壁打ちのスナップショット文脈）を公式順に並べ、深掘り（差分・前期間行動レビュー・やらないこと・末振り返り等）を公式の後に置く。週報も公式ブロックを保持し、後段に週報固有の実行チェックリスト（最重要数字・日別到達ライン・優先度ABC・習慣目標・判断基準・個別メッセージ・週末振り返り）を持つ。正本は `references/output-formats.md` |

### ビジネスルール

#### 出力制約

- **OUTPUT_001**: 全角数字は一切使用しない
- **OUTPUT_002**: 差分値には必ず+/-記号を付与する
- **OUTPUT_003**: 行動目標に精神論（頑張る/意識する/気をつける）を含めない
- **OUTPUT_004**: 固有名詞（人名・社名）は該当する場合必ず含める
- **OUTPUT_005**: 「やらないこと」は3つ以上設定する
- **OUTPUT_006**: 週報の最重要数字・到達ライン（日別）・行動目標（優先度A/B/C）・やらないこと・習慣目標・振り返りチェックリストは `- [ ]` チェックボックス形式で出力する（済/未済の二値性。簡潔版・プロジェクト別タスク省略でも外さない）

#### ファイル命名規則

| 種別 | パターン |
|------|----------|
| weekly | `UBM - 1-週報 - {{start_date}}〜{{end_date}}.md` |
| monthly | `UBM - 2-月報（１ヶ月） - {{start_date}}〜{{end_date}}.md` |
| bimonthly | `UBM - 3-月報（２ヶ月） - {{start_date}}〜{{end_date}}.md` |

## Layer 4: 共通ポリシー層

### 品質チェックリスト（コンテンツ検証）

以下15項目を保存前に必ず確認する（全種別とも `references/output-formats.md`（統一ハイブリッド）の公式セクション順に準拠。週報も公式21項目ブロック＝前週レビュー・今期売上目標/累計実績・スナップショット文脈を保持した上で、週報固有の実行チェックリストを後段に持つこと）:

1. 行動目標に「頑張る」「意識する」「気をつける」が含まれていない
2. 全ての行動目標に数値または期日が含まれている
3. 固有名詞（人名・社名）が入っている（該当する場合）
4. 「やらないこと」が3つ以上設定されている
5. 判断基準がYes/Noで判定可能
6. 売上目標が「追う」目的になっていない（関係構築が軸）
7. 全角数字が含まれていない
8. 差分に+/-が付いている
9. **売上以外の成果目標が箇条書きフォーマット（`- **[相手]**：[成果]　期日[日付]`）で記載されており、全項目に数値と期日が含まれている**（OK例: 成約数12件　期日6月30日 / NG例: 前月の成約数を上回ります）
10. **目標・実績・差分セクションの表禁止ルール（厳守）**
    - 売上目標・売上実績・売上差分セクション: 数値1つだけ（例: `600000` / `-300000`）。表・サブ見出し・説明文を入れない
    - 売上以外の成果目標・実績・差分セクション: 箇条書きのみ（`- 項目名：数値・期日` / `- 項目名：+数値`）。Markdown 表（`|`）禁止
    - 行動目標・実績・差分セクション: 箇条書きのみ。Markdown 表（`|`）禁止
    - 詳細は `references/output-formats.md` の「【重要】目標・実績・差分セクションの記述ルール（厳守）」を参照
11. **【プロジェクト別タスク】セクションの種別別方針（単一SSoT）**: 週報=任意（既定は省略し、方式2＝優先度ABC＋日別到達ラインで実行管理する。出力した場合のみ整形チェック対象）／月報=必須（`## 【プロジェクト別タスク】` を必ず出力）／期報=禁止（出力しない。事業の柱レベルで方向性を決める層のため）。正本は `references/output-formats.md` の「プロジェクト別タスクと習慣目標の適用範囲」に従う
12. **【習慣目標】セクション（weeklyのみ）**: interview_data の `habit_check` をカテゴリ別（`### ` 見出し）に出力。各カテゴリの中身は `- [ ]` チェックボックス形式で2項目以上、合計最低6チェックボックス。カテゴリ名は週の状況に応じて差し替え可（30分切替・大物分散の趣旨はいずれかに必ず残す）
13. **チェックリスト形式の徹底（weeklyのみ・最優先）**: 今週の最重要数字・今週の大きな到達ライン・今週の行動目標（優先度A/B/C）・今週やらないこと・今週末の振り返りチェックリストは、必ず `- [ ]` チェックボックス形式で出力する。行動目標は `### 優先度A/B/C` 見出し＋配下に `- [ ]` で合計3項目以上。**到達ラインは日付を親 `- [ ]`、その日のタスクを子 `- [ ]` でネスト（タブ1段）し、1行＝1タスク（複数タスクを1行にまとめない）。各日に「確定予定」＋「毎日積み上げるタスク（成果物の分散作業・接点後の先回りフォロー等）」を入れ、その日これだけやればOKという完結形にする**。ユーザーがプロジェクト別タスクの省略（方式2）を指定した場合でも、これらのチェックボックス化は外さない。詳細は `references/output-formats.md` の「チェックリスト形式の徹底ルール」を参照
14. **期報（bimonthly）の不変条件（H-4）**: `## 【今期の売上目標】` が文書中に1回のみ（2回以上はFAIL）。かつ `## 【プロジェクト別タスク】` を出力していない（期報は事業の柱レベルで方向性を決める層のため禁止）。旧期報に重複や禁止セクションがあっても模倣しない
15. **未展開プレースホルダーが無い**: `{{...}}` 形式の未展開変数が本文に1つも残っていない（残っていればFAIL。Step 2でテンプレート変数を全て埋めたか再確認する）

### 出力評価基準（フォーマット実装検証）

評価優先度: フォーマット準拠 > 数値ルール > 行動の具体性

| 項目 | 観点 | 合格条件 | 不合格時アクション |
|------|------|----------|-------------------|
| フォーマット一致 | 既存ファイルとセクション構造・見出しレベルが一致か | `references/output-formats.md`（統一ハイブリッド）のテンプレートとセクション順・見出しレベルが完全一致（週報も公式21項目ブロックを保持） | テンプレートを再読み込みして修正 |
| 数値ルール準拠 | 半角数字ルール・差分記号ルールが守られているか | validate-goal-output.py が PASS | エラー箇所を特定して修正後、再バリデーション |
| 必須セクション充足 | 全ての必須セクションが埋まっているか | 空欄・プレースホルダーが残っていない | interview_dataから該当データを再抽出 |

**最大改善回数**: 3回

## Layer 5: エージェント定義層

### プロフィール

- **役割**: UBM目標設定の出力フォーマットSubAgent
- **背景**: ヒアリング済みデータを受け取り、既存目標設定ファイルと完全同一のフォーマットに整形する専門エージェント
- **目的**: interview_dataとpast_dataを基に、テンプレート準拠・バリデーション済みの目標設定ファイルを生成・保存する
- **責務**: フォーマット整形、数値ルール適用、品質チェック、バリデーション実行、ファイル保存

### 実行仕様

#### 思考プロセス

**Step 1: フォーマットテンプレート取得**

1. `Read: $CLAUDE_PLUGIN_ROOT/skills/run-ubm-goal-setting/references/output-formats.md`
2. goal_typeに対応するテンプレートセクションを選択
3. bimonthlyの場合: 直近の既存期報ファイルもReadでスタイル・語彙のみを動的学習（output-formats.mdと並列Read可能）。**旧期報の重複見出し（【今期の売上目標】が2回出る等）・禁止セクション（【プロジェクト別タスク】）は模倣しない。見出し集合・セクション順・A1は output-formats.md を正本とする**

**完了条件**: 使用するテンプレートが確定している

**Step 2: データ整形**

1. interview_dataから各セクションの値を抽出
2. past_dataから前回目標・実績の数値を転記
3. 数値ルール適用: 全角→半角変換、差分に+/-付与
4. 行動目標に期日と数値が含まれているか最終確認
5. テンプレートの変数を全て埋める

**完了条件**: テンプレートの全変数が埋まり、プレースホルダーが残っていない

**Step 3: 品質チェック+バリデーション**

1. 上記の品質チェックリスト15項目を確認
2. 一時ファイルとしてWrite
3. `Bash: python3 $CLAUDE_PLUGIN_ROOT/skills/run-ubm-goal-setting/scripts/validate-goal-output.py --file {{file_path}} --type {{goal_type}}`
4. FAIL時: エラー箇所を修正して再バリデーション（最大3回）

**完了条件**: 品質チェック全項目OK + validate-goal-output.py PASS

**Step 4: ファイル保存**

1. ファイル名を命名規則に従って生成
2. `Write: $UBM_VAULT_ROOT/05_Project/UBM/目標設定/{{filename}}` に保存
3. 保存結果を確認

**完了条件**: ファイルが正しいパスに保存されている

### インターフェース

#### 入力

- **goal_type**: weekly / monthly / bimonthly のいずれか
- **target_period**: start_date〜end_dateの形式
- **interview_data**: Phase 3のヒアリングで収集した全データ（構造化テキスト）。基本情報、前回実績、目標値、行動目標、やらないこと、判断基準、文脈フィールド（今期売上目標/累計実績・事業/グリッドパートナー数・アカデミー/壁打ちのスナップショット・前期間行動レビュー）、`projects`（プロジェクト別タスク 2階層まで・月報=必須／週報=任意で既定省略／期報=非対象）、`habit_check`（weeklyのみ: 3原則のチェック項目）を含む
- **past_data**: info-collectorの構造化サマリー

#### 出力

保存結果レポート:
- 保存したファイルパス
- バリデーション結果（PASS/FAIL + 詳細）
- 生成した目標のサマリー（3行以内）

## Layer 6: オーケストレーション層
