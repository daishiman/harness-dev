# 北原孝彦 原則データベース

UBMナレッジから抽出した北原さんの原則・名言集。
目標設定の各フェーズで適切に引用するためのデータベース。

## カテゴリ別原則

### 関係構築（最重要 - 全フェーズ共通）

> 「売上目標は追わない。人との関係を育むことが一番大事」

適用: **全目標設定時の最上位原則**。売上は結果であり、追うものではない。行動目標の軸は常に「人との関係を育む」行動にする。

> 「相談をもらうことがすべて」

適用: 無形ビジネスの行動目標設定時。外交の目的は「売る」ことではなく「相談をもらう」こと。

> 「自分の利益を最後にしろ。相手の利益を先にしろ」

適用: 商品設計・価格設定の目標設定時。顧客視点の確認。

> 「信頼できる人を探すな。信頼される人になれ」

適用: パートナーシップ・チーム構築の目標設定時。

> 「届けないことのほうが不親切」

適用: 前提検証（Step 3）。「迷惑かもしれない」という前提の破壊。

**軌道修正の判断基準**:
目標が以下のパターンになっていたら、関係構築ファーストに軌道修正する:
- 「月商◯◯円を達成する」が目的化 → 「◯◯さんとの関係を深め、相談を受ける」に修正
- 「成約◯件」が先頭に来ている → 「接点◯件→面談◯件→結果として成約」の順序に修正
- 行動目標に「人の名前」が入っていない → 具体的な誰かとの関係構築に落とし込む

### 思考→感情→行動の連動（目標設定の根幹）

> 「何を考えて、どう思って、どう行動するかを考える」

適用: **行動目標設定の根幹**。行動を設計するとき「なぜその行動をするのか（考え）」「相手は何を思うか（感情）」「だから何をするか（行動）」の3層で目標を組み立てる。数字だけの行動目標ではなく、相手の感情と自分の思考が見える目標にする。

> 「人が何を考えてどう動くか、自分以外の周りの感情を理解しないといけない」

適用: 外交・営業目標の設計時。「メッセージを3件送る」ではなく「相手が何を求めているかを理解した上で、相手が喜ぶ情報を3件届ける」に変える。

> 「相手の感情を理解して、成果を出す」

適用: 行動の質を高める際。量の目標（◯件）に加えて、「相手の立場で考えた行動か？」のチェックを入れる。

**目標設定への落とし込み方**:
行動目標を設計する際、以下の3問で深掘りする:
1. **考え**: なぜその行動をするのか？（自分の思考の整理）
2. **思い**: 相手は何を思い、何を求めているか？（相手の感情の理解）
3. **行動**: だから具体的に何をするか？（思考と感情を踏まえた行動）

例:
- NG: 「個別メッセージを3件送る」（行動だけ）
- OK: 「佐藤さんは4月のセミナー集客で困っている（思い）。自分のAI活用ノウハウが役立てる（考え）。だから佐藤さんにAI集客の事例3つをまとめて送る（行動）」

### 行動管理

> 「行動を管理する。成果や他人の評価はコントロールできない」

適用: 行動目標の設定時。成果目標ではなく行動目標に焦点を当てる。

> 「迷う＝止まる。迷う状況を作らない」

適用: 「やらないこと」の設定時。判断基準を明確にして迷いを排除。

> 「目標設定したものとズレた振り返りを行わない」

適用: 最終確認（Step 5）。設定した目標と振り返りの整合性確認。

### 外交・営業

> 「商品を作ってから外交する。商品なしでの外交は無意味。ただし、商品があるなら届けて届けて届けていく」

適用: フェーズ判定後。0→1で商品がない場合は商品作りが先。ある場合は外交に全力。

> 「接触方法と回数を決めて、市場に何度も何度もアプローチする」

適用: 行動計画（Step 4）。外交の具体的な行動設計。

### マインドセット

> 「3ヶ月本気でやれば0→1は当たり前のように達成できる」

適用: 0→1フェーズの目標設定時。期間と覚悟の基準。

> 「自分のための涙は1人で流せ。人のための涙は堂々と流せ」（PR-032）

適用: 指摘や感情に揺れているユーザーへ。感情をコントロールして人前に出る経営者の在り方を伝える際に使用。自分への矢印ではなく、相手への矢印を持つことの重要性と組み合わせて引用する。

> 「挑戦の平均回数は0.8回。つまり人は1回も本気で挑戦していない」（PR-035）

適用: 「やってみたが上手くいかない」「挑戦したがダメだった」という発言をするユーザーへ。「本当に本気で挑戦しましたか？」と問い返す根拠として使用。行動量・質・継続性を再定義させる際に引用。起業を迷っている・やめようとしているユーザーへの覚悟の確認にも有効。

### 選択と集中

> 「今のフェーズで一番やるべきことに集中する」

適用: 前提検証+目標設定（Step 3）。フェーズ別の注力領域の明確化。

> 「やることを決めるのではなく、やらないことを決める」

適用: 「やらないこと」セクションの設定時。

> 「当たり前のことは当たり前に、先にやるべきことは先にやるべき」

適用: 新規施策より基盤整備を優先させたい場面。新しいことに飛びつく前に、Googleマップ整備・口コミ収集など当たり前の基盤を先に完成させる指導に使用。

### 報告・コミュニケーション

> 「感情が入ると盛り上がってしまう。社員や部下と話す際にも、常に事実ベースの状況確認と、そこに対する打開策を話すようにしている」

適用: ユーザーが感情的に状況を話したり、報告の質が低い場合の軌道修正。行動目標の振り返り時にも事実ベースで整理させる際に使用。

> 「答えを出しているのではない。成果が出るための姿勢を整えている」（PR-033）

適用: コンサル・支援の場面で「答えを教えてほしい」という姿勢になっているユーザーへ。自分で考える力を育てることの重要性を伝える際に使用。指示待ちや依存体質からの脱却を促す文脈でも引用。

### 信頼・人間関係

> 「信じてくれた人を勝たせることを最優先にする」

適用: 活動が散らかっている、優先順位が決まらないユーザーへ。「信じてくれた顧客・クライアントを勝たせることを軸に優先順位を整理させる」際に使用。

> 「お金とは神（価値）が循環するから入ってくるもの。現金だけがお金ではない」

適用: 「売上が上がらない」「お金がない」と焦っているユーザーへ。価値の循環という視点で、現在している行動の意味を再解釈させる際に使用。

> 「ビジネスモデルは一時的なハック。原理原則こそが再現性を持つ」

適用: 新しいビジネス手法やトレンドに飛びつきがちなユーザーへ。「今やっていることの原理原則は何ですか？」と問い返す際に引用。

### 選択と集中（追加）

> 「やりたいことを商品にするな。欲しいと言われるものを商品にする」

適用: 商品設計・ビジネスモデル設計時。自分の興味ではなく、顧客の需要から商品を設計させる際に使用。

> 「体験を経験に変えることで恐怖がなくなる」

適用: 行動を先延ばしにしている、「準備が足りない」を理由に踏み出せないユーザーへ。「今怖いことは体験していないから。小さく体験することで恐怖は消える」という転換に使用。

### 顧客理解・コンセプト設計

> 「なぜ紹介されましたか、が答えであり、なぜ継続していただけてるんですか、がコンセプト」

適用: 集客やブランディングに悩むユーザーへ。コンセプトを自己定義しようとしているユーザーに、既存顧客へのヒアリングを促す際に使用。

> 「シャンプーもトリートメントも、在庫が3か月で履ける状態のお客さんが揃っていたから作った。システムも作った瞬間に300店舗導入の見込みがあった状態だった」（PR-036）

適用: 新商品・新サービスを作ろうとしているユーザーへ。特に広告投資前に需要確認ができていない場合。「商品を作りました。どう売りますか？」という相談に対し、「先にお客様が揃っていますか？揃ってから作るのが正しい順序です」と問い直す際に使用。

### 事業戦略・資本保全

> 「利益を傷つけないこと、資本を減らさないこと。負けというのは、資本や利益を大きく傷つける負け方。だから最初は汗をかく、会いに行く、案件を取りに行く、紹介を取りに行くという戦い方を必ず入れておく」（PR-037）

適用: 広告投資を検討しているユーザー、または既存の施策が資本を消費しているユーザーへ。「広告をやりたい」という相談に対し、「まず広告なしで売れる施策から試しましょう。打率の高いものが分かってから広告に移行が正しい順序です」と誘導する際に使用。

### 組織・リーダーシップ

> 「社長が自由でいいように見えてはいけない。社長だけが自由で社員だけが縛られる状態では不信感につながる」

適用: 組織のルール整備や社員との信頼関係に悩む10→100フェーズのユーザーへ。社長がルールの模範を示すことの重要性を伝える際に使用。

> 「社長がぶれると組織全体が弱くなる。生き様を見せることが最強の組織構築法」（PR-034）

適用: 組織が不安定・バラバラな方向を向いているユーザーへ。社長の在り方・一貫した姿勢がそのまま組織文化になることを伝える際に使用。採用・人材育成に悩む10→100フェーズのユーザーへも有効。

> 「採用で最初に聞くのは『あなたはどこまで逃げずに仕事ができますか？』です。スキルより逃げない人かどうかが全て」（PR-038）

適用: 採用基準をスキル・経験値で判断しているユーザーへ。「どんな人材を採用すればよいか」という相談に対し、スキルより「逃げない人格・当事者意識・利益責任」を基準にするよう転換させる際に使用。採用した人材がすぐ辞める・困難時に逃げるという問題を抱えているユーザーへも有効。

> 「マインドじゃない、思いなんだ。テクニックの中に思いを入れてください」（PR-039）

適用: テクニック訴求だけで仲間・パートナー・採用がうまくいかないユーザーへ。「仲間が集まらない」「採用がうまくいかない」という相談に対し、「なぜこれをやっているのか、誰の問題を解決したいのか。その思いをテクニックの中に入れてください」と問いかける際に使用。

> 「利益相反は100%起きる前提で、同業者を顧客にする市場設計をしなさい」（PR-040）

適用: 同業者支援に踏み切れないユーザー、ノウハウ開放の線引きに悩むユーザーへ。「同業者に教えると競合が増える」という懸念を持つユーザーに「利益相反は前提として受け入れ、その上で同業者を顧客にできる設計とは何ですか？」と問いかける際に使用。

> 「事業は一枚の絵から組み立てなさい。川上を押さえてから川下を任せる」（PR-041）

適用: 低価格帯から始めて伸び悩んでいるユーザー、事業全体の構造が見えていないユーザーへ。「どの商品から始めるべきか」という相談に対し、「まず事業を一枚の絵で見てください。川上（高単価・上流）は何ですか？そこを先に設計して逆算しましょう」と問いかける際に使用。

> 「チームに売上目標を落とすな。ビジョンを落とせ。売上は後からついてくる」（PR-042）

適用: チームが売上目標に向かって動かないユーザー、施策に追われて組織の方向性がバラバラになっているユーザーへ。「なぜチームが動かないか」という相談に対し、「今チームにビジョンを語れていますか？報酬もビジョンと連動していますか？」と問いかける際に使用。

> 「人がついてくる人間かどうか。それが一番大切な基準です。大義を掲げ、思想を語り、ビジョンを見せる」（PR-043）

適用: 組織化・採用に苦戦しているユーザーへ。「人が集まらない」「採用がうまくいかない」という相談に対し、採用の前に「あなたは人がついてくる人間ですか？大義・思想・ビジョンを語れていますか？」と自己点検させる際に使用。

## フェーズ別の適用ガイド

### 全フェーズ共通（最重要）

| 状況 | 適用する原則 |
|------|-------------|
| 売上目標を追い始めている | 「売上目標は追わない。人との関係を育むことが一番大事」 |
| 行動が「売る」に偏っている | 「相談をもらうことがすべて」 |
| 合宿アドバイスと目標がズレている | 合宿で決めた方向性を再提示して軌道修正 |
| 目標と振り返りがズレる | 「目標設定したものとズレた振り返りを行わない」 |
| 指摘で感情的になる・傷つく | 「自分のための涙は1人で流せ。人のための涙は堂々と流せ」（PR-032） |
| 答えを求めすぎる・依存体質 | 「答えを出しているのではない。成果が出るための姿勢を整えている」（PR-033） |
| 「やった・挑戦した」が言い訳化している | 「挑戦の平均回数は0.8回。本当に本気で挑戦しましたか？」（PR-035） |
| 商品を作ってから売ろうとしている | 「先にお客様が揃っている状態を作ってから商品を出す（オーディエンスファースト）」（PR-036） |
| 広告・SNS投資を最初の手段にしている | 「まず汗をかく施策から。打率が分かってから広告へ。資本を傷つけない戦い方を最優先に」（PR-037） |

### 0→1フェーズ

| 状況 | 適用する原則 |
|------|-------------|
| 商品が未完成 | 「商品を作ってから外交する」 |
| 外交を躊躇 | 「届けないことのほうが不親切」 |
| 成果が出ない焦り | 「3ヶ月本気でやれば0→1は当たり前」 |
| 何をすべきか迷う | 「迷う＝止まる。迷う状況を作らない」 |
| 売上ばかり気にする | 「まず1人に全力で価値提供。売上は後からついてくる」 |

### 1→10フェーズ

| 状況 | 適用する原則 |
|------|-------------|
| 外交が足りない | 「接触方法と回数を決めて、何度もアプローチ」 |
| 価格で悩む | 「自分の利益を最後にしろ」 |
| 多角化したくなる | 「今のフェーズで一番やるべきことに集中」 |
| 成約率が低い | 「相談をもらうことがすべて」 |
| 数字ばかり追っている | 「関係を育むことが先。売上は結果」 |

### 10→100フェーズ

| 状況 | 適用する原則 |
|------|-------------|
| 人材採用で悩む | 「信頼できる人を探すな。信頼される人になれ」 |
| 組織化で迷う | 「やることではなく、やらないことを決める」 |
| チームの関係が希薄 | 「人との関係を育むことが一番大事」 |
| 組織の報連相が乱れる | 「常に事実ベースの状況確認と打開策を話す」 |
| 社長だけが例外になっている | 「社長だけが自由で社員だけが縛られる状態では不信感につながる」 |
| 独立志向の人材への対応 | 「キャリアを見せる、ビジョンを語る、一緒にやらないかと語る」 |
| 組織がバラバラな方向を向いている | 「社長がぶれると組織全体が弱くなる。生き様を見せることが最強の組織構築法」（PR-034） |
| 採用基準がスキル・経験値になっている | 「スキルより逃げない人かどうかが全て。採用で最初に確認するのは逃げない姿勢」（PR-038） |

## 使い方

1. ヒアリング中、ユーザーの発言に対応する原則があれば自然に引用する
2. 「北原さんもこう言っていますね」の形で添える
3. 押し付けるのではなく、気づきのきっかけとして使う
4. 1回の対話で引用は1〜2回まで（多すぎると説教臭くなる）
5. UBMナレッジファイルに最新の学びがあれば、このDBより優先する
