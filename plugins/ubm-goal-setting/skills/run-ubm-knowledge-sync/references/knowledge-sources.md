# UBM ナレッジソースガイド

UBM目標設定で参照するナレッジファイルの取得方法と優先順位。

---

## ナレッジ同期システム

`05_Project/UBM/` 配下の情報は日々更新される。
最新の情報をスキルに反映するため、以下の仕組みで動的に管理する:

```
[情報の流れ]

05_Project/UBM/（情報ソース: 日々追加・更新）
  ↓ detect-knowledge-updates.py で差分検知（MD5ハッシュ）
  ↓ /ubm-knowledge-sync コマンドで抽出・分類
knowledge/*.json（内容別JSON: 原則/相談/フェーズ/行動/転換/事例）
  ↓ knowledge/router.json でルーティング
  ↓ info-collector が必要カテゴリだけ読み込み
Phase 3 ヒアリング（ユーザー状況に最適な情報+北原さんの表現でアドバイス）
```

**ナレッジ管理コマンド（Claude Codeから実行）:**
```bash
# 更新検知（差分のみ）
! python3 $CLAUDE_PLUGIN_ROOT/skills/run-ubm-knowledge-sync/scripts/detect-knowledge-updates.py --registry $CLAUDE_PLUGIN_ROOT/knowledge/registry.json --sources $UBM_VAULT_ROOT/05_Project/UBM

# 全件再構築
! python3 $CLAUDE_PLUGIN_ROOT/skills/run-ubm-knowledge-sync/scripts/detect-knowledge-updates.py --registry $CLAUDE_PLUGIN_ROOT/knowledge/registry.json --sources $UBM_VAULT_ROOT/05_Project/UBM --all

# ナレッジ同期（検知→抽出→JSON格納を一括実行）
/ubm-knowledge-sync
```

**新しい情報を追加したら `/ubm-knowledge-sync` を実行してスキルに反映する。**

## ナレッジディレクトリ構成

```
05_Project/UBM/
├── 目標設定/              ← 過去の目標ファイル（Phase 1で使用）
├── 動画教材/              ← UBM公式教材（Tier 1: 最重要）
├── YouTube/               ← 北原さんのYouTube議事録（Tier 1.5: 事業相談パターンが豊富）
├── 合宿/                  ← 合宿での個別フィードバック（Tier 1: 方向性決定の最重要ソース）
├── 月報フィードバック/     ← 月報フィードバック会の議事録（Tier 2: 具体的指摘）
├── minutes - *.md         ← セミナー・イベント議事録（Tier 3）
├── UBM - *.md             ← セミナー・勉強会・成果報告（Tier 3）
└── *.pdf                  ← ワークシート（参考）
```

## 優先順位別ナレッジマップ

### Tier 1: 必ず参照（方向性決定の最重要ソース）

| ファイル/ディレクトリ | 内容 | 使う場面 |
|----------|------|----------|
| `合宿/`（**最新ディレクトリ**） | 合宿での個別事業相談・方針決定 | **全目標設定時**（方向性の軸） |
| `動画教材/UBM - vol.1_商品づくりロードマップ.md` | 商品設計の手順 | 0→1フェーズで商品がない場合 |
| `動画教材/UBM - vol.2_商品販売ロードマップ.md` | 販売・外交の手順 | 外交・集客の目標設定時 |
| `動画教材/UBM - vol.3_関連商品づくりロードマップ.md` | 関連商品・LTV向上 | 1→10以降の商品拡充時 |
| `動画教材/UBM - 選択と集中.md` | フェーズ別の注力領域 | 戦略的思考の適用時 |

### Tier 1.5: YouTube議事録（事業相談パターンの宝庫）

| ファイル | テーマ | 推奨使用場面 |
|----------|--------|-------------|
| `YouTube/`（**インデックス経由**） | 北原さんの事業相談議事録 | ユーザー状況に類似する事例があるとき |

**YouTube議事録ファイル別マッチングガイド:**

| ファイル名 | テーマキーワード | 使う場面（Phase 3 のStep） |
|-----------|----------------|--------------------------|
| `2025-09-21 - 迷ったら川上を見ろ.md` | 選択と集中, 事業の核心, 意思決定 | Step 3: 何をすべきか迷っている場合 |
| `2025-10-28 - ビジネスモデル根本的見直し.md` | 事業転換, ピボット, 方向修正 | Step 2: 前回と同じパターンを繰り返している場合 |
| `2025-10-29 - 目標がブレる起業家へ告ぐ...経営者思考の真髄.md` | 目標ブレ, 経営者思考, 一貫性 | Step 3: 毎回目標を変えるパターンの場合 |
| `2025-11-05 - ...人が集まってくる思考を叩き込む.md` | 外交, 関係構築, 信頼, 自己中心 | Step 2: 外交が少ない/迷っている場合 |
| `2025-11-19 - 有望な若手がパンクしてしまいました.md` | 行動量過多, 優先順位, 燃え尽き | Step 2: 行動はしているが成果が出ない場合 |
| `【企業努力】法律の間を潜り抜ける勝ち筋.md` | 勝ち筋, 戦略, 差別化, 法律 | Step 3: 10→100フェーズの戦略検討時 |

**取得戦略（コンテキスト効率重視）:**
1. まず `knowledge/router.json` → 該当カテゴリの `knowledge/*.json` を参照（最優先）
2. ナレッジが空（未同期）の場合 → ファイル名一覧から上記テーブルでマッチング
3. 合致ファイルは最大2本。Grep で関連キーワード検索 → 前後30行を抽出（全読みしない）
4. 2〜9万文字の大型ファイルなので、先頭200行 + 目次（`^#` Grep）→ 関連セクションのみ精読

### Tier 2: ビジネス形態・状況に応じて参照

| ファイル | 内容 | 使う場面 |
|----------|------|----------|
| `動画教材/UBM - 無形ビジネス講座.md` | 無形ビジネス戦略 | 無形ビジネスの場合 |
| `動画教材/UBM - 店舗ビジネス講座.md` | 店舗ビジネス戦略 | 店舗ビジネスの場合 |
| `動画教材/UBM - グリッドパートナーセミナー.md` | パートナーシップ戦略 | 10→100フェーズ |
| `月報フィードバック/`（**最新2件**） | 月報FB会での北原さんの具体的指摘 | 直近の改善点とパターン把握 |

**注意**: `vol.3_関連商品づくりロードマップ.md` は1→10以降のフェーズでのみ参照。
0→1フェーズでは不要。月報フィードバックは週報では最新1件、月報・期報では最新2件を読む。

### Tier 3: 補足情報（直近3ヶ月以内のファイルは必ず確認）

| ファイル/ディレクトリ | 内容 | 使う場面 |
|-------------|------|----------|
| `minutes - *.md`（UBMルート直下） | 合宿オリエン・月報FB会等のセミナー議事録 | 特定テーマの深掘り時 |
| `UBM - *.md`（UBMルート直下） | X活用術・交流会攻略・成果報告会等 | 特定テーマのアドバイス時 |

**取得ルール**:
- Glob: `$UBM_VAULT_ROOT/05_Project/UBM/UBM - *.md` と `$UBM_VAULT_ROOT/05_Project/UBM/minutes - *.md`
- ファイル名の日付で降順ソートし、直近3ヶ月以内のファイルは必ず先頭100行を読む
- 目標設定に関連するテーマ（セミナー内容、イベント記録）があれば構造化サマリーに含める

## 過去目標ファイルの取得ルール

### 取得元

`05_Project/UBM/目標設定/`

### ファイル名パターン

| 種別 | パターン |
|------|----------|
| 週報 | `UBM - 1-週報 - YYYY-MM-DD〜YYYY-MM-DD.md` |
| 月報 | `UBM - 2-月報（１ヶ月） - YYYY-MM-DD〜YYYY-MM-DD.md` |
| 期報 | `UBM - 3-月報（２ヶ月） - YYYY-MM-DD〜YYYY-MM-DD.md` |

### 目標種別ごとの取得スコープ

| 作成する目標 | 取得する過去情報 |
|-------------|------------------|
| 週報 | 直前の週報1件 + 現在の月報 |
| 月報 | 直近1ヶ月の全週報 + 現在の期報 + 前回月報 |
| 期報 | 直近2ヶ月の全週報 + 全月報 + 前回期報 |

### 日付抽出ロジック

ファイル名の末尾 `YYYY-MM-DD〜YYYY-MM-DD` から期間を抽出。
`〜` または `~`（全角・半角両方）に対応すること。
最新ファイルはファイル名ソートで取得可能（日付が先頭にあるため）。

## ナレッジ検索の実行手順

### 0. ナレッジルーター参照（最優先）

```
Read: $CLAUDE_PLUGIN_ROOT/knowledge/router.json
→ quick_lookup.by_phase / by_issue でマッチング
→ 該当カテゴリの knowledge/*.json を Read
```

### 1. ファイル一覧の取得

```
Glob: $UBM_VAULT_ROOT/05_Project/UBM/目標設定/UBM - {1,2,3}-*.md
Glob: $UBM_VAULT_ROOT/05_Project/UBM/動画教材/*.md
```

### 2. 関連ナレッジの選定

ユーザーのフェーズ・形態に応じて Tier 1 → Tier 2 の順に読み込む。
Tier 3 は具体的なアドバイスが必要な場合のみ。

### 3. 読み込み効率化

- 大きなファイルは `limit` パラメータで先頭200行を読む
- 目次・見出しから関連セクションを特定し、そこだけ読む
- 同じファイルを2回読まない（1回読んだら要約をメモ）
