# UBM目標設定 高速ヒアリングテンプレート

目標種別ごとの最適化された質問フロー。
過去データから自動取得できる項目はスキップし、最短ルートで情報収集する。

---

## 設計原則

- **1ターン1〜3問**: ユーザーが答えやすい分量
- **自動取得優先**: 過去ファイルから取れる情報は聞かない
- **バッチ質問**: 関連する質問はまとめて聞く
- **段階的深掘り**: 浅い回答には1回だけ深掘り、それ以上は次へ進む
- **所要時間目安**: 週報5〜8分 / 月報10〜15分 / 期報15〜20分
- **並列Read活用**: Phase 3開始時の参照ファイル4本は並列Readする

## 週報（1週間目標）フロー

### ターン1: 先週の振り返り確認
```
先週の週報を確認しました。

[自動表示: 先週の目標サマリー]

先週の目標について、実績はどうでしたか？
1. 売上実績（数字）
2. 行動目標の達成状況（何件やったか）

困ったことや想定外だったことがあれば教えてください。
```

### ターン2: 差分分析 + 原因
```
[差分を自動計算して表示]

この差分について、一番大きな原因は何だと思いますか？
（1つだけ挙げてください）
```

### ターン3: 今週の目標設定
```
[前回の原因分析を踏まえて]
[アカデミーアドバイスがある場合: アカデミーで決めた方向性を提示]

では今週について:
1. 最重要数字（今週追う数字1つ）は何にしますか？
2. 大きな到達ライン（今週の最も重要なゴール）は？

※ 北原さんの言葉: 「売上目標は追わない。人との関係を育むことが一番大事」
→ 数字を追うのではなく、「誰との関係を育むか」を軸に考える
```

### ターン4: 行動計画
```
[目標を踏まえて]

今週の具体的な行動目標を決めましょう:
北原さん:「何を考えて、どう思って、どう行動するかを考える」

1. 毎日やること
   NG:「個別メッセージを3件送る」（行動だけ）
   OK:「佐藤さんはセミナー集客で困っている→AI集客の事例をまとめて送る」
   →「誰が」「何を求めているか」「だから何をするか」の3点で設計
2. 今週中にやること（期日+具体的成果物）

それぞれ「考え（なぜやるか）」「思い（相手は何を求めているか）」
「行動（誰に・何を・いつまでに・何件）」で教えてください。
※ 行動目標には具体的な人の名前を入れると解像度が上がります
```

### ターン5: やらないこと + 判断基準
```
2つだけ:
1. 今週「やらない」と決めること（迷いを減らすため・3つ以上）
2. 迷ったときの判断基準（1文で）

例: 「売上に直結しないことはやらない」
例: 「人との接点が増えることか？ Yes→やる / No→やらない」
```

### ターン5.5: プロジェクト別タスク（粗い棚卸し）
```
抱えているプロジェクトを2階層まで（プロジェクト→必要ならサブ）で粗く出してください。
各プロジェクトには先方担当者を入れてください。各タスクは「期日」「提出先・宛先」「対象物・行動」だけでOK。

形式:
### ▼ プロジェクトA：{{name}}（目的：{{why}}）
- 先方担当者: {{client_owner}}
- [ ] [{{YYYY-MM-DD}}] [{{提出先・宛先}}] {{対象物・行動}}

1. プロジェクト名と目的（1文）を全て
2. 各プロジェクトの先方担当者（未確認なら未確認）
3. 各プロジェクトの今週分のタスクを期日・提出先付きで（粗くてOK）

※ 自主開催AI勉強会・外部AI勉強会登壇は毎回同じ定型チェックリストで生成します。全項目が完了したら、翌月分として新規作成します。
```

### ターン5.7: 習慣目標の運用パラメータ（週報のみ）
```
毎日の運用を支える習慣目標を確定します。
1. 1日に回す30分ブロックは最低何本？
2. 今週分散させる「大物タスク」と、何日に分けるか？

※ 30分切替・分散・粗い粒度の3原則は固定。週末にチェックボックスで振り返ります。
```

### ターン6: アカデミー整合性 + 関係構築チェック → 確認 → 出力
```
[完成した週報を表示する前に以下をチェック]

【アカデミーアドバイスとの整合性】
※ アカデミーデータが[自動取得]の場合のみ実行。[要ヒアリング]の場合はスキップ
※ [鮮度注意]（3ヶ月超）の場合は参考情報扱い
直近のアカデミー（{{camp_date}}）で決めた方向性と照らし合わせます:
- 方向性: OK / 要修正
- 具体的アクションの反映: OK / 未反映

【関係構築チェック】
- 行動目標に「人の名前」が入っていますか？
- 「売上を追う」ではなく「関係を育む」行動になっていますか？
- 「相談をもらう」「接点を増やす」が軸になっていますか？

[整合性チェック後、完成した週報を表示]

この内容で保存してよいですか？
修正したい箇所があれば教えてください。
```

```

### ターン5: 今月の売上目標
```
今月（{{期間}}）の売上目標はいくらにしますか？（半角数字）

[確率思考の適用]

- メッセージ {{X}} 件 → 面談 {{Y}} 件 → 成約 {{Z}} 件

3. 北原さんの原則で今月特に意識すべきことは？

[ナレッジから適切な原則を提案]
```

```

### ターン2: 前回実績のヒアリング
```
前回の2ヶ月（{{期間}}）について:

```

→ [回答に応じて1回深掘り] その課題の根本にあるものは？

### ターン4: 事業の柱 + 行動変化
```

### ターン5: 戦略ロードマップ
```

## スマートスキップルール

過去データから自動取得できる場合にスキップする項目:

| 項目 | スキップ条件 |
|------|-------------|
| 名前 | 前回ファイルに記載あり |
| 事業内容 | 前回ファイルに記載あり（変更確認のみ） |
| フェーズ | 前回ファイルから推定可能 |
| 形態 | 前回ファイルに記載あり |
| 顧問参加前月商 | 前回ファイルに記載あり |
| 前回の目標値 | 前回ファイルから自動取得 |
| 合宿日程 | 合宿ディレクトリから推定可能 |

## 回答パターン別の対応

詳細は `agents/phase3-coordinator.md` の「回答パターン対応」セクションを参照。

基本方針:
- 具体的数字 → 次へ
- 曖昧 → 数字で再質問
- 精神論 → 仕組みに転換
- 沈黙 → 3択提示
- 長文 → 1文要約確認
