# 行動目標 ベストプラクティス

過去の目標設定ファイルから抽出した「良い行動目標」のパターン集。
目標設定時にこの基準で具体性を担保する。

---

## 売上以外の成果目標のフォーマット（月報・期報）

### フォーマット定義

```
- **[相手/項目]**：[数値化された成果・状態]　期日[日付]
```

### 実例

```
- **日工さん**：研修実施1回・売上75,000円　期日4月中旬
- **森田さん**：単価合意1件・業務委託開始1件　期日4/2
- **海老名さん**：単価決定1件・業務委託開始1件　期日4/15
- **阿部さん**：Claude Code+GWS提案1件　期日4/10
- **田村さん**：GWS提案方向性整理1件　期日4月中
- **Google Workspace**：議事録自動化パイプライン構築1件　期日4/5
- **AI質問会勉強会**：自主開催1回・参加者3名以上　期日4/18
- **合宿**：参加・実績報告1回　期日4/25
- **成約数**：新規成約12件　期日6/30
```

### 記述ルール

| ルール | OK例 | NG例 |
|--------|------|------|
| 数値化必須 | 成約数12件　期日6月30日 | 前月の成約数を上回ります |
| 期日必須 | 単価合意1件　期日4/2 | 単価合意を目指す |
| 状態（達成基準）を明記 | 業務委託開始1件 | 業務委託が始まっている（状態の記述のみ） |
| 相手名は固有名詞 | **森田さん**：... | **クライアント**：... |

### 注記

> 前月の未達に対する行動変化を踏まえた上であなたが約束する売上以外の成果目標（期日と状態）を数値化して記載すること

## 週報の行動目標パターン

### 良い例（実際の過去ファイルから抽出）

```
* 阿部さんとの打ち合わせを実施し、4月の関東出張時の対面候補日を打診する
* 2026-03-24に佐藤さん、海老名さんとの打ち合わせを実施し、ヒアリング内容をもとに価値提供できる内容を共有する
* 森田さん経由案件は、日程確定または次回化を進める
* 手塚さんとの接点は、初回顔合わせの日程化または面談実施を進める
```

### パターン分析

| 要素 | 具体例 | 必須/推奨 |
|------|--------|-----------|
| **誰に** | 阿部さん、佐藤さん | 必須（固有名詞） |
| **何を** | 打ち合わせを実施、候補日を打診 | 必須（動詞で終わる） |
| **いつまでに** | 2026-03-24に、今週中に | 必須（日付または期限） |
| **成果物/ゴール** | 日程確定、次回化 | 推奨（完了条件が明確） |
| **数値** | 個別メッセージ3件/日 | 繰り返し行動には必須 |

## 月報の行動目標パターン

### 構成要素

月報の行動目標は以下の3層構造:

1. **日次行動**（毎日やること）
   - 例: 「個別メッセージを1日3件送る」
   - 例: 「毎朝9時にSNS投稿1本」

2. **週次行動**（毎週やること）
   - 例: 「週1回の勉強会を開催する」
   - 例: 「週2件の面談を実施する」

3. **期限付きタスク**（今月中にやること）
   - 例: 「3/15までにセミナー資料を完成させる」
   - 例: 「3月中にA社との契約を確定させる」

### 数字の入れ方

| パターン | 例 |
|----------|-----|
| 件数/日 | 個別メッセージ3件/日 |
| 件数/週 | 面談2件/週 |
| 件数/月 | 勉強会4回/月 |
| 累計 | 月内に新規接点20件 |
| 金額 | 契約300000円を確定 |
| 期日 | 3/15までに完了 |

---

## 期報の行動目標パターン

月報の3層構造 + 以下を追加:

4. **マイルストーン**（2ヶ月の中間地点）
   - 例: 「1ヶ月目: A社契約確定 + 新規2社との接点作り」
   - 例: 「2ヶ月目: 月商600000円の達成」

5. **戦略的行動**（フェーズ進行に関わる行動）
   - 例: 「商工会議所との連携体制を構築する」
   - 例: 「助成金申請を完了する」

## 「やらないこと」の書き方パターン

### 良い例（過去ファイルから抽出）

```
* 売上や案件前進に直結しない新規開発
* 細部を詰めすぎて全体の前進が止まること
* 資料を完璧にすること自体を目的にすること
* 面談や登壇を、その場限りで終わらせること
```

### パターン: 「〜すること」で統一

- 動詞の名詞化で終わる
- 具体的な行動を否定する（抽象的な「ダメなこと」ではない）
- 北原さんの「選択と集中」に基づく

---

## 判断基準の書き方パターン

### 良い例（過去ファイルから抽出）

```
今週の終わりに見るべきものは、頑張った感ではない。

* 次回の約束が増えたか
* 新しい接点が増えたか
```

### パターン: 「〜したか？」の疑問形

- Yes/No で判断できる基準
- 数値で測定可能
- 「頑張った感」を排除する視点

## 合宿アドバイスとの整合性チェック

### チェック項目

目標設定時に、直近の合宿フィードバックと以下の点で整合性を確認:

1. **方向性の一致**: 合宿で決めた事業方針と目標がズレていないか
2. **フェーズの一致**: 合宿で確認したフェーズに合った目標か
3. **具体的アドバイスの反映**: 合宿で受けたアドバイスが行動目標に落とし込まれているか
4. **前提の更新**: 合宿で覆された前提が目標に反映されているか

### チェック方法

```
直近の合宿（{{camp_date}}）で受けたアドバイス:
[合宿ファイルから抽出]

今回の目標と整合性を確認:
- 方向性: OK / 要修正
- フェーズ: OK / 要修正
- アドバイス反映: 反映済み / 未反映（具体的に何が未反映か）
```

## 品質チェックリスト

目標設定完了前に以下を確認:

- [ ] 行動目標に「頑張る」「意識する」「気をつける」が含まれていない
- [ ] 全ての行動目標に数値または期日が含まれている
- [ ] 固有名詞（人名・社名）が入っている（該当する場合）
- [ ] 「やらないこと」が3つ以上設定されている
- [ ] 判断基準がYes/Noで判定可能
- [ ] 直近の合宿アドバイスと方向性が一致している
- [ ] 売上目標から逆算した行動量が明記されている
- [ ] 北原さんの原則との整合性がある
