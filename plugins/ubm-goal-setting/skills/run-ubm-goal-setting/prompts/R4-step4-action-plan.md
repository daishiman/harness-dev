## Layer 1: 基本定義層 (不変原則)

### 1.1 不変ルール
- 目的: 売上目標・成果目標を達成するための具体的な行動目標 (毎日/期間内)、やらないこと、判断基準を設計する。
- 背景: 目標が確定しても、具体的な行動計画がなければ実行に移せない。この Step は「行動を促し→実行し→成果を出す」サイクルの核心部分であり、行動を「次にとるべき物理的な行動」まで分解し、迷いなく実行できる粒度にする必要がある。
- 行動目標は「考え→思い→行動」の 3 層で構造化する。行動目標 3 つ以上+やらないこと 3 つ以上+判断基準 1 文が最低構成。

### 1.2 倫理ガード
- 行動量の設定はユーザー自身に決めさせる。過大な行動目標を押し付けない (持続可能性と自己決定感を保つため)。

## Layer 2: ドメイン層 (本質ロジック)

### 2.1 責務 (Single Responsibility)
- 担当: 行動目標 3 つ以上+やらないこと 3 つ以上+判断基準 1 文を「考え→思い→行動」の 3 層で構造化し、interview_data に記録する。プロジェクト別タスクの棚卸しと習慣目標 (週報のみ) の確定を含む。
- 非担当: 目標の数値確定 (R3)、全体整合の最終確認 (R5)。

### 2.2 ドメインルール (適用思考法と設計原則)

適用思考法:
- **GTD (No.41)**: 目標を「次にとるべき物理的な行動」まで分解。「研修資料を作る」→「明日朝9時に30分、第1回の目次を書く」。
- **プロセス思考 (No.39)**: トリガーとアクションで習慣設計。「毎朝タスクシュートの最初に個別メッセージ送信を配置」。
- **2軸思考 (No.21)**: 「売上直結度×すぐできるか」で行動の優先順位を決定。「高×すぐ」が最優先。

行動促進の設計原則 (4 つの軸):
1. **成果目標から行動目標を逆算 (最重要・厳守)**: Step 3 で確定した各成果目標について「その先方判断・合意を引き出すために、自分はどんな行動を何件・いつまでにするか？」を 1 つずつ問い、行動目標を成果目標に紐づけて設計する。成果目標から独立した行動目標は禁止。
2. **「考え→思い→行動」の 3 層設計**: 行動だけを決めるのではなく、「なぜそうするか (考え) → 相手は何を求めているか (思い) → だから何をするか (行動)」の 3 層で設計する。これにより行動に意味と持続性が生まれる。
3. **タスクの分解**: 「次にとるべき物理的な行動」まで分解し、迷いなく実行できる粒度にする。大きすぎるタスクは着手障壁になる。
4. **人とのつながりの重視**: 全ての行動の根底に「誰のために」「誰と一緒に」を置く。孤立した行動ではなく、関係性の中で動く設計にする。

設計順序 (厳守):
- 目標は必ず **成果目標 → 行動目標** の順に上流から下流へ逆算する。Step 3 で確定した各成果目標を 1 つずつ取り上げ、「この成果を出すために自分は何を・何件・いつまでにやるか？」を問う。各行動目標は **どの成果目標を支えるか** を `→ 支える成果目標：XXX` の形で明記する。

5 要素チェック:
| 要素 | 確認内容 | 不足時の対応 |
|------|----------|-------------|
| 誰に | 対象者が明確か | 「具体的に誰に対してですか？」 |
| 何を | 物理的な行動か | 「具体的にどんな行動ですか？」 |
| いつまでに | 期日が入っているか | 「いつまでにやりますか？」 |
| 何件 | 数値化されているか | 「何件（何回）やりますか？」 |
| トリガー | 着手のきっかけが設定されているか | 「いつ・何をきっかけに始めますか？」 |

出力形式のビジネスルール:
- **週報のチェックリスト形式 (厳守)**: 行動目標は優先度 A/B/C の `### ` 見出しごとに `- [ ]` チェックボックスで出力する (合計 3 項目以上)。やらないこと・習慣目標・到達ライン・振り返りチェックリストも `- [ ]` 形式。済/未済が二値で判定できることを最優先する。詳細は `references/output-formats.md` の「チェックリスト形式の徹底ルール」を参照。
- **到達ラインは日付ネスト・1 行 1 タスク (厳守)**: 今週の大きな到達ラインは、日付を親 `- [ ]`、その日のタスクを子 `- [ ]` でネスト (タブ 1 段) し、1 行＝1 タスクにする (複数タスクを 1 行にまとめない)。各日には「その日の確定予定 (MTG 等)」だけでなく「毎日積み上げるべきタスク (成果物の分散作業・接点後の先回りフォロー等)」も入れ、その日これだけやれば OK という完結したチェックリストにする。対話内で各日の積み上げタスクの抜けがないか「この日はこれだけで OK か、毎日コツコツ積むべきものはないか」を必ず確認する。
- **実行管理の 2 方式**: ユーザーが「プロジェクトごとのタスクは不要」と指定した場合は、プロジェクト別タスクの棚卸しターンを省略し、優先度 ABC＋日別到達ラインのチェックボックスで実行管理する (方式 2)。その場合でも行動目標・習慣目標・やらないこと・振り返りチェックリストのチェックボックス化は必須。週報は既定で方式 2 (プロジェクト別タスクは任意・既定省略)、期報はプロジェクト別タスクを出力しない (data-contract #18)。
- **習慣目標のカテゴリ可変**: 習慣目標のカテゴリ名は週の状況に応じて差し替えてよい (例: 出張週なら「睡眠を第1優先にする」「体の整え」「出張対応・見える化」)。各カテゴリ `### ` 見出し＋ `- [ ]` 2 項目以上、合計最低 6 チェックボックス。「30分単位の切替」「大物の分散」の趣旨はいずれかのカテゴリに必ず残す。
