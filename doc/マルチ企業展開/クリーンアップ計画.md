# クリーンアップ計画 — 不要物の削除準備 Runbook

この文書は、harness 正本化の前段として、リポジトリ内の「`xl-skills` の古いコピー」「旧構造の残骸」を安全に削除できる状態を作るための Runbook である。削除対象の分類・判定根拠・実行手順・検証ゲート・ロールバックを定める。**実削除はユーザー承認後に本書 §2 の手順でのみ行う。**

位置づけ: [移管計画.md](移管計画.md) §5「削除・保持 disposition」の実行子文書。disposition の語義と安全原則([移管計画.md](移管計画.md) §9)は移管計画を正本とし、本書は具体対象と手順だけを持つ。

## 5. ロールバック

| 対象 | 復元方法 |

## 6. 禁止事項

[移管計画.md](移管計画.md) §9 を正本とし、本書固有分を加える。

- §1-C の stale 部分コピー(`plugins/` 等)を本書の手順で個別削除しない(移管計画 Step 2-4 と同一手順でのみ入れ替える)。
- `git clean -fdx` を repo root に対して直接実行しない(§9 再掲)。
- 削除前に §2 の事前チェック(原本存在 + 未追跡確認)を省略しない。
- `doc/マルチ企業展開/` と `eval-log/` の elegant-review 成果物を削除対象に含めない。
