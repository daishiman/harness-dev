# 取引基本契約書 対話式作成スキル

7人の専門家による多角的レビューを経て、法的リスクを最小化した取引基本契約書を自動生成する。

## 概要

| 項目 | 内容 |
|------|------|
| **目的** | ユーザーとの対話を通じて情報収集し、法的・税務的リスクを最小化した取引基本契約書を生成 |
| **出力形式** | Markdown形式（PDF変換可能） |
| **対応業種** | AI事業、IT事業、コンサルティング、デザイン等 |
| **法令準拠** | 民法（2020年改正）、フリーランス新法（2024年）、著作権法、下請法 |

## 期待される成果物

1. **取引基本契約書**（Markdown形式）
2. **法的リスク分析レポート**
3. **税務チェックリスト**（印紙税・源泉徴収・インボイス）
4. **AI利用リスク評価**（該当する場合）

## 成功基準

- ユーザーの事業内容・取引条件に合致した契約書
- 準委任契約・請負契約の両方に対応
- AI利用時のリスク免責条項を含む（AI事業の場合）
- 著作権条項に**第27条・第28条**を明記
- フリーランス新法（支払期日60日以内等）に準拠
- 反社会的勢力排除条項を含む

### 条件分岐

- **AI利用なしの場合**: AI・データ法務スペシャリストをスキップ
- **税務判断が複雑な場合**: 税務アドバイザーへ再要求
- **法務判断が複雑な場合**: 法務リサーチャーへ再要求

### 反復実行

- ユーザーからの修正要望に応じて、関連エージェントを再実行
- 品質管理者 → 契約書ドラフター → リスク分析官 → 品質管理者のループ
- **最大反復回数: 5回**

### 制約

- **必須経由**: 情報収集アナリスト, 契約書ドラフター, 品質管理者
- **完了判定**: 成功基準の全項目を満たし、ユーザーが最終確認OKを出した時点で完了

## ベストプラクティス

| すべきこと | 避けるべきこと |
|-----------|---------------|
| 法令条文は番号を明記して引用 | 法的有効性の保証 |
| 不確実事項は「可能性があります」等の限定詞使用 | 税務申告代行 |
| 最新法令はWebSearchで確認 | 訴訟戦略の提案 |

# レイアウト検証
pnpm cli layout 契約書.md

pnpm cli tax input.json

# ヘルプ
pnpm help
